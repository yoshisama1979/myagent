# OVERVIEW — このリポジトリは何か

> 新しいスレッドの開始時や、全体像を見失ったときに読むファイル。
> CLAUDE.md には「最低限の核」だけを置き、肉付けはここに集約する（毎ターン読まれる CLAUDE.md を軽く保つため）。

---

## 一言で

**株式会社はなさかの社長が、経営分析・戦略を AI と一緒に考えるための作業環境。**

ここにある成果物（`site/` の記録、`_assistant/` のチャット等）は、すべて
**社長（人間）が理解・判断しやすくするための補助ツール** という位置づけ。
「AI に何かを自動で運用させるシステム」ではなく、「社長の思考と判断を支えるための足場」。

- **主語は常に社長（人間）**。AI は右腕・壁打ち相手であって、主役ではない。
- `site/` … 社長が**見やすいように**記録を HTML で残す場所（Nginx + Tailscale で閲覧）。
- `_assistant/` … **どこからでも（スマホ含め）確認できる**ようにしたチャット入口。

## ディレクトリ早見表

| パス | 何か | 備考 |
|------|------|------|
| `CLAUDE.md` | プロジェクト規範（毎ターン読まれる核） | 詳細はこの OVERVIEW と `rules/` に外出し |
| `SYSTEM.md` | **動くモード同士の地図**（役割・一次情報源・承認ゲート・ハンドオフ） | 統括者の入口。OVERVIEW=ディレクトリ地図／SYSTEM=モード地図 |
| `site/` | ブラウザ閲覧用 HTML 全般（記録・トラッカー・スキルシート等） | Nginx ドキュメントルート。Tailscale 経由のみ公開 |
| `_assistant/` | 経営サポート用チャットアプリ（FastAPI、**読み取り専用**） | 詳細は `_assistant/README.md` / `_assistant/PLAN.md` |
| `bin/` | 財務・案件・スキルシートの**生成スクリプト群**（Python） | 詳細は `bin/README.md`。`hana-api.sh` は hana-tools 連携 |
| `data/` | 財務・機密データ（生成スクリプトの入出力ソース） | **ブラウザ非公開**。`secrets/` あり、取り扱い注意 |
| `rules/` | 協働・運用ルール（partnership / memo / development 等） | CLAUDE.md から参照される運用ルール本体 |
| `ai-loop/` | Reviewer AI ↔ Dev AI の**非同期レビュー基盤** | `conversation.md` 経由。`/reviewer-cycle` で起動 |
| `etc/` | プレビューサーバ設定（Nginx / PHP-FPM の conf） | 構成は `rules/preview-server.md` 参照 |
| `.claude/` | Claude Code 設定・スキル・ロール別ルール | `reviewer.md` / `memo.md` 等のモード定義 |
| `claude-rules/` | **このシステムとは無関係。** 弊社がクライアント向けシステムを作る際の**汎用ルールテンプレ**置き場（**複数プロジェクトで共有・各案件にコピーして使う**） | `website/` `websystem/`。**固有値（エージェントID・サイト名・公開URL・DB名・カラー等）を本文に焼き込まない**＝各プロジェクトの `project-config.md` に置き、ルール本文はそれを参照する形で書く（汎用ルールと固有値の分離）。更新時もこの分離を保つ |

## ルールファイルの設計方針（今後の指針）

ルール・ドキュメントを追加するときは、次の二層に分ける：

1. **誤解防止の核（3〜4行）** … 読まなくても間違えないための最小限。CLAUDE.md に直接埋め込む。
2. **肉付け（詳細）** … 必要なときだけ読みに行く参照ファイルに外出しし、CLAUDE.md からはリンクで誘導する。

CLAUDE.md は毎ターン丸ごとコンテキストに載るため、**太らせない**こと。
「リンクだけ」にもしない（タスクに必要と判断されないと読まれないため、核は必ず埋め込む）。
