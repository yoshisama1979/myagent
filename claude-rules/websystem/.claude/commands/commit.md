プロジェクトルート（project-config.md 参照）で `git diff --staged` を実行し、ステージングされた差分を確認してください。

問題がある場合（バグ・仕様違反・セキュリティリスク・不要な変更）はコミットメッセージを提案せず、問題点を指摘してください。

問題がない場合は、差分に関しての日本語でコミットメッセージを見出しと項目で提案してください。
コミットメッセージは `.claude/rules/commit.md` の規約（**`[領域]` プレフィックス必須**・領域分割・50/72）に準拠する。過去ログのスタイルが規約から明確に外れている場合は、rules/commit.md「エージェントへの指示」の確認手順（過去ログに合わせるか規約を適用するか）に従う。
過去のコミット履歴（git log --oneline -5）のフォーマットに合わせること。
