# 環境固有の設定メモ（雛形）

## 目的

このファイルは、**このマシン・このプロジェクト固有**の実行方法・シェルの癖・回避策を記録する場所。
テンプレート時点では空（雛形のみ）。各環境で運用しながら記入していく。

汎用的なルール（全プロジェクト共通の規約）はここに書かない — それは他のルールファイルの領分。

## 記入する内容の例

- **言語ランタイムの実行方法**（例: Python は `python3` 直呼びか、venv/conda 経由か、コンテナ経由か）
- **シェル環境の注意**（例: Git Bash / WSL / PowerShell の使い分け、UNCパスで失敗するコマンド、エンコード関連の癖）
- **うまく動かない時の調査手順を残す作法**：
  - **推測で進めず原因を調査**する（エラーメッセージ・使用シェル・パスの種類を確認）
  - 恒久的に実行方法を変更した場合は、**このファイルを更新**してナレッジを残す

## 記入例の型

````markdown
## ○○の実行方法

（前提・環境の説明を1〜2行）

### コマンド例

```bash
# 実行例
<コマンド>
```

### 注意事項

- （失敗するパターンとその理由）
- （代替手段）
````
