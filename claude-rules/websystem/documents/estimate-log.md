# 見積もり台帳（estimate-log）

**採用された（受注・顧客承諾済みの）見積もり**の記録先。次回の見積もり作成時に、類似案件の工数感・項目立て・但し書きの参考にする（`rules/modes/quotation.md` §6 準拠）。

**記録のゲート（最重要）**：採用されたかどうかは AI には観測できない（受注はリポジトリ外の出来事）。**AI が自己判断で記録しない**。見積もり提示後、**ユーザーに「採用されたか・台帳に記録してよいか」を確認し、承認された場合のみ**追記する。

## 記録フォーマット

各エントリは通し番号 `EST-NNN` で追記する（append-only・過去エントリの本文は書き換えない。後日更新してよいのは「状態」「実績」欄のみ）：

- **EST-NNN**: 一行要約（対象機能）
- **日付**: 見積もり提示日／採用確認日
- **対象**: 何の改修・機能追加か（関連仕様書・goals.md へのリンク）
- **原本**: 提示した見積書ファイルのパス・版（あれば）
- **項目と工数**: 標準6項目ベースの内訳（人日 or 時間）
- **金額**: 記載はユーザーの判断（記録時に確認。書かない場合は「非記載」。書く場合は通貨・税込/税別を明記）
- **但し書きの要点**: スコープ外・運用制約・検収条件など、効いた条項
- **状態**: 採用 / 改訂あり（後継 EST-NNN）/ 取消 / 完了（後日更新可）
- **後継・関連**: 減額改訂・追加見積もり・取消は旧エントリを書き換えず**新しい EST を起こして相互参照**する
- **実績**（後日・任意）: 完了後の実工数・見積もりとの差と要因。次回の精度向上の材料
- **記録承認**: ユーザー確認日（誰がいつ「採用・記録OK」と言ったか）
