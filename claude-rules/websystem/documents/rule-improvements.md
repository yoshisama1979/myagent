# ルール改善メモ（rule-improvements）

ルールファイル（`CLAUDE.md` ・`.claude/rules/` 配下のルール定義 等）について **気づいた改善点・気になった点** の記録先。
`CLAUDE.md`「コラボレーションの原則 2. ルールファイル自体の改善も提案する」準拠。

**ユーザーが明確な回答をしなかった場合も、気づきはその都度ここに記録する**（記録と反映を分離）。
提案が会話に埋もれて流れ、忘れられるのを防ぐための仕組み。反映はユーザー判断のタイミングでまとめて行う。

## 記録フォーマット

各エントリは以下を明記する（通し番号 `RI-NNN` で連番）：

- **対象**: どのファイルの、どの記述か
- **気づき**: 何が気になったか（実態との乖離／抜け／矛盾／構成・粒度）
- **どう変えるか**: 改善案
- **なぜ**: 根拠
- **状態**: 未反映 / 反映済み（反映済みは削除してよい）
- **還元**: `[テンプレ還元]` を付ける（このプロジェクト固有でなく、**はなさか共通テンプレ `claude-rules/` 全体に還元すべき**と判断したとき）／付けない（このプロジェクト内で閉じる話。多くは `project-config.md` 側の調整）
- **還元状態**（`[テンプレ還元]` のときだけ）: `持ち帰り待ち`（既定＝棚卸しで社長が渡す） → （渡したら）`持ち帰り済み` → （反映されたら削除）。mailbox で送った場合は `送信済み(M-ID)`／受け口未整備なら `受け口未整備・要社長連絡`。**この状態を見て二重連絡・取りこぼしを防ぐ**

反映が済んだエントリは削除する（または「反映済み」と印を付ける）。

### `[テンプレ還元]` 印のものをテンプレ更新側へ届ける

`[テンプレ還元]` と判断した気づきは、テンプレート更新側（はなさかAI）へ届けるとテンプレ自体が良くなる。**このテンプレ（websystem）は拠点横断メールボックスを前提にしない**ので、既定の届け方は次のとおり：

- **人手キャリーバック（既定）**：`[テンプレ還元]` 印のまま残しておき、**棚卸しのタイミングで社長がまとめて myagent（テンプレ更新側）へ渡す**。この受信箱に自動で流す仕組みは持たない。**＝ best-effort（社長が棚卸しでこのファイルを見て初めて届く）**。取りこぼしたくない重要な還元は、この印のまま放置せず、社長に一言直接伝えるか mailbox（下記）で送る。
- **mailbox が配線済みのプロジェクトなら（任意）**：`bin/mailbox.sh` と `.env`（`MAILBOX_URL`/`MAILBOX_TOKEN`）が用意されている場合は、website と同じく `bash bin/mailbox.sh send --to rule-kaizen --type request --thread websystem-kaizen --subject "[websystem/<ファイル名>] 要旨" "対象／観察／提案／根拠"` で送ってよい（受け口 `rule-kaizen` 未整備なら空振りせず、ここに残して社長に伝える）。

いずれも **プロジェクト固有の値（実パス・スタックのバージョン・アカウント種別等）は持ち込まず**、汎用的なルールの直しだけを還元する（固有値は `project-config.md` 側）。反映は myagent 側の棚卸しで社長承認を得てから行われる。
