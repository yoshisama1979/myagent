# 事業ファクト集（company-facts）— コンテンツの情報 SSOT

このファイルは、**サイトに載せる「事業の確定事実」の単一の真実の源（SSOT）**。
`Design.md` が「デザインの SSOT」であるのに対し、これは **「コンテンツ（事実情報）の SSOT」**。
（`web-hanasaka` からの現場提案をテンプレ化。2026-07-03）

## なぜ必要か

- ヒアリング等で確定した事実（料金・技術対応範囲・実績・提供方針・法的表記 等）を **会話履歴に置くとセッションを跨いで消える**（AIエージェント特有の制約）。
- 公開HTMLは **「公開用に間引いた結果」** なので、HTMLだけを見ても「まだ書いていない」「そもそも存在しない」「**あえて書かないと判断した**」の区別がつかない。この曖昧さが、再ヒアリングや捏造・誤掲載のリスクになる。
- だから確定事実は、**ページに書く前にまずここへ記録**し、コンテンツはここから展開する（**情報整理 → 展開** の順）。「あえて公開しない」という判断も **確定事項として残す**（残さないと将来のセッションで再提案・誤掲載が起きる）。

## 使い方（rules/modes/page-build.md / rules/modes/page-update.md と連携）

- `rules/modes/page-build.md` §5・`rules/modes/page-update.md` §5「数値・仕様・固有情報は一次情報源で裏取り」の **一次情報源の恒久的な置き場がここ**。ページに直接書く前に、まずファクト集に記録してから展開する。
- `rules/modes/page-build.md` §6・`rules/modes/page-update.md` §6「要確認情報」の **確認が取れた確定事実の記録先もここに一本化**する。ヒアリング/クライアント確認で新事実が出たら都度追記する。
- 各項目には **公開可否** と **出典** を必ず付ける（下記フォーマット）。
- ヒアリング（`rules/modes/hp-loop-dialogue.md` 経由の社長確認・素材依頼を含む）で確定した事実は、回答が返ってきたらここへ昇格させる。

## 記録フォーマット

各ファクトは通し番号 `F-NNN` で、以下を必須にする：

- **事実**: 確定した内容（数値・固有名詞・仕様はそのまま）
- **公開可否**（3択のいずれか）:
  - `公開済み`（掲載先の URL / ページを併記）
  - `公開可・未掲載`（載せてよいが、まだ載せていない）
  - `非公開`（**あえて載せない判断**。理由を添える＝この判断自体を確定事実として残す）
- **出典**: 誰が・いつ確認したか（例：社長ヒアリング 2026-07-03 ／ クライアント〇〇様 メール）
- **備考**: 補足・関連ファクト・失効条件（期間限定の価格・改定予定 など）

> **未確認・要確認のものはここに「確定事実」として書かない**（`rules/modes/page-build.md` / `rules/modes/page-update.md` §6 のとおり、確定するまでは載せない）。確認が取れたらここへ昇格する。

## カテゴリ（例。プロジェクトに合わせて取捨）

| カテゴリ | 例 |
|---|---|
| 会社・事業概要 | 社名・所在地・沿革・事業内容 |
| サービス・商品 | 提供内容・特徴・技術対応範囲 |
| 料金・費用 | 価格・料金体系・オプション |
| 実績・成果数値 | 導入数・実績値・受賞歴 |
| お客様の声・事例 | 掲載可否は要確認 |
| 法的表記 | 特商法・免責・資格・許認可番号 |
| その他固有事実 | 上記に収まらない確定事実 |
