# ルール改善メモ（rule-improvements）

ルールファイル（`CLAUDE.md` ・`.claude/rules/` 配下のルール定義 等）について **気づいた改善点・気になった点** の記録先。
`CLAUDE.md`「コラボレーションの原則 2. ルールファイル自体の改善も提案する」準拠。

**ユーザーが明確な回答をしなかった場合も、気づきはその都度ここに記録する**（記録と反映を分離）。
提案が会話に埋もれて流れ、忘れられるのを防ぐための仕組み。反映はユーザー判断のタイミングでまとめて行う。

## 記録フォーマット

各エントリは以下を明記する（通し番号 `RI-NNN` で連番）：

- **対象**: どのファイルの、どの記述か
- **気づき**: 何が気になったか（実態との乖離／抜け／矛盾／構成・粒度）
- **どう変えるか**: 改善案
- **なぜ**: 根拠
- **状態**: 未反映 / 反映済み（反映済みは削除してよい）
- **還元**: `[テンプレ還元]` を付ける（このプロジェクト固有でなく、**はなさか共通テンプレ `claude-rules/` 全体に還元すべき**と判断したとき）／付けない（このプロジェクト内で閉じる話。多くは `project-config.md`／`Design.md` 側の調整）
- **還元状態**（`[テンプレ還元]` のときだけ）: `未送信` → `送信済み(M-ID)` → （反映されたら削除）。受け口未整備で送れなかったものは `受け口未整備・要社長連絡` と書く。**この状態を見て二重送信・取りこぼしを防ぐ**（`送信済み` を再送しない・`未送信`/`受け口未整備` を放置しない）

反映が済んだエントリは削除する（または「反映済み」と印を付ける）。

### `[テンプレ還元]` 印のものはテンプレ更新側へ送る

`[テンプレ還元]` と判断した気づきは、**テンプレートを更新する側（はなさかAI）へ届ける**とテンプレ自体が良くなる。送り方は [`.claude/rules/mailbox.md`](../.claude/rules/mailbox.md)「2-3. テンプレ改善提案を送る（`rule-kaizen` 宛）」に従う（`bash bin/mailbox.sh send --to rule-kaizen …`）。mailbox 未設定・受け口未整備なら送らず、ここに残して社長に伝える（記録は消さない）。
