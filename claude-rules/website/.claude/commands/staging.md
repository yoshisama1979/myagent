変更した内容をステージングしてください。

## 注意

- `git add .` や `git add -A` を**使わない**。必要なファイルだけ個別に `git add <path>` する
- **`.scss` 編集時のコンパイル済み `.css` の扱いはプロジェクト方針に従う**。既定は **`.css` を一緒にステージしない**（古い／二系統コンパイラの差分を混ぜないため、`.scss` のみをステージ）。
  - 自動コンパイル環境（`.claude/rules/scss-autocompile.md`）で `.css` をコミットに含める運用なら、生成元を一本化した上で対象 `.css` を明示的にステージする。迷ったら `.scss` のみに留めてユーザーに確認する
- `images/` 配下の大きなファイルや、`*.BAK` `index_org.html` 等のバックアップを誤ってステージしていないか確認する

## 完了後

ステージしたファイル一覧を `git status` で報告すること。
