`git diff --staged` を実行し、ステージングされた差分を確認してください。

## 差分の評価

以下に該当する問題があれば、コミットメッセージを提案せず、問題点を指摘してください。

- バグ・仕様違反
- セキュリティリスク(XSS の余地、入力サニタイズ漏れ、ハードコードされた認証情報等)
- 不要な変更(デバッグ用 `console.log` / コメントアウトされた古いコード等)
- **コンパイル済み `.css` の直接編集**(本プロジェクトでは `.scss` のみ編集する運用)
- **画像・バイナリの誤コミット**(過大サイズ、未圧縮、`*.BAK` 等)
- Design.md からの逸脱(未定義の色トークン・新規ブレークポイント等)

## コミットメッセージの提案

問題がなければ、日本語でコミットメッセージを **見出し1行 + 必要に応じて箇条書きの本文** で提案してください。

### フォーマットの基準

- 過去のコミット履歴(`git log --oneline -10`)のスタイルに合わせる
- 見出しは1行・50文字以内目安・体言止めまたは「〜の追加 / 更新 / 修正」
- 本文(任意): 大きな変更がある場合のみ、何を変えたか・なぜ変えたかを箇条書きで
