まず `rules/modes/refactor.md` を読み込んでガイドラインを確認してください。

その後、`git diff --staged` でステージング済みの差分を確認し、ガイドラインに従ってリファクタリング候補を**列挙してください**。

## 重要

- **その場で書き換えない**。候補を提案して、ユーザーの承認を1つずつ得てから着手する
- 提案フォーマットはガイドラインの「Step 2. 提案を列挙する」セクションに従う
- SCSS を触る変更を提案する場合は「開発者側でコンパイル」のステップを完了報告に含めること
- 動いている古いページ(project-config.md のレガシー命名パターンに該当するもの。例: `*_org.html`)への変更提案は最小限に

ステージング差分が空の場合は、ユーザーに対象範囲(直近のコミット / 特定ファイル / 全体)を確認してください。
