# ウェブサイト完成後チェックルール

ウェブサイトの制作が一通り完了した段階で、納品・公開前に**品質を担保するためのチェック項目**をまとめたルール。

## 基本方針

- 「動いている」状態と「品質が担保された」状態は別物として扱う
- このチェックは**実装完了後、公開前**に通すゲートと位置付ける
- すべての項目を機械的に通す必要はないが、**該当する項目は明示的に確認したか／問題なかったかを報告**する
- 問題が見つかったら、**修正候補をユーザーに提示**してから手を入れる（勝手にリファクタしない）

---

## 実施手順

### 1. チェック対象の特定

- 今回作成・変更したページ、CSS、画像、JavaScript を `git status` / `git diff` で確認する
- ページ単位でチェック範囲を決める（複数ページある場合は1ページずつ）

### 2. 各カテゴリのチェック実施

下記「チェック項目」セクションを上から順に確認する。

### 3. 結果報告

以下のフォーマットでユーザーに結果を報告する。

```
## チェック結果

### ✅ 問題なし
- [カテゴリ名] [項目名]

### ⚠️ 要確認 / 要修正
- [カテゴリ名] [項目名]: [具体的な内容と該当ファイル・行]
  - 修正案: [どう直すか]

### ⏭️ スキップ
- [カテゴリ名] [項目名]: [スキップ理由]
```

### 4. 修正は許可を得てから

要修正項目に対しては、ユーザーの承認を得てから修正に着手する。一括で直さず、項目ごとに合意を取りながら進めることが望ましい。

---

## チェック項目

### A. 画像

- [ ] **ファイルサイズ**: 1枚あたり概ね 200KB 以内（メインビジュアル等大きな画像でも 500KB 以内が目安）
- [ ] **画像形式の選択**:
  - 写真 → JPG（または WebP）
  - ロゴ・アイコン・線画・透過必要 → PNG または SVG
  - イラスト・図形 → SVG が望ましい
  - 装飾的な大きな画像 → WebP を検討
- [ ] **解像度**: 実際の表示サイズの 2倍程度（Retina対応）に収まっているか。表示サイズに対して過剰に大きい画像が放置されていないか
- [ ] **`width` / `height` 属性**: `<img>` タグに指定されているか（CLS 対策）
- [ ] **`alt` 属性**: すべての `<img>` に意味のある `alt` が設定されているか。装飾画像は `alt=""` で明示
- [ ] **ファイル名**: 半角英数・ハイフン区切り。日本語・スペース・全角記号を含まない
- [ ] **配置場所**: プロジェクトのルールに沿ったディレクトリに配置されているか（例: `images/`, `img/`, `assets/images/`）
- [ ] **未使用画像**: HTML/CSS から参照されていない画像ファイルが残っていないか

### B. CSS / Sass

- [ ] **インラインスタイル**: `style="..."` 属性を使っていないか（緊急時を除き禁止）
- [ ] **`!important` の使用**: 多用していないか。詳細度で解決できるはず
- [ ] **セレクタの書き方**:
  - タグセレクタ単体（`ul`, `a`, `h1` 等）への直接指定をしていないか（リセット用途を除く）
  - ID セレクタの過剰使用がないか
  - 過度に深いネスト（4階層以上）がないか
- [ ] **命名規則**: プロジェクトのルール（BEM 等）に沿っているか
- [ ] **重複ルール**: 同じ要素に対する同じプロパティが複数箇所で定義されていないか
- [ ] **未使用クラス / 未使用 CSS**: HTML から参照されていない CSS ルールが残っていないか
- [ ] **メディアクエリ**: ブレークポイントが統一されているか（マジックナンバーが散らばっていないか）
- [ ] **ベンダープレフィックス**: 必要に応じて付与されているか（Autoprefixer 等を使っているなら設定確認）
- [ ] **コンパイル後の CSS**: Sass を使う場合、コンパイル後の CSS が最新の状態か（古いコンパイル結果が残っていないか）。自動コンパイル環境（`scss-autocompile.md`）が有効なら編集ごとに最新化されるが、念のため `.scss` と `.css` の整合を確認。無ければ開発者のコンパイル後に確認

### C. HTML

- [ ] **DOCTYPE / lang**: `<!DOCTYPE html>` と `<html lang="ja">` が正しく設定されているか
- [ ] **`<meta charset>`**: `UTF-8` が指定されているか（`<head>` の最上部）
- [ ] **`<meta name="viewport">`**: モバイル対応の viewport 指定があるか
- [ ] **`<title>`**: ページごとに適切なタイトルが設定されているか（サイト名と区切り文字込み）
- [ ] **`<meta name="description">`**: 各ページに固有の説明文が設定されているか（120文字前後）
- [ ] **見出し階層**: `<h1>` がページに1つ。`<h2>` → `<h3>` の順序が崩れていないか
- [ ] **セマンティックなタグ**: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>` を適切に使えているか（`<div>` だらけになっていないか）
- [ ] **リンク切れ**: `<a href="...">` のリンク先が存在するか。`#` のままになっていないか（プレースホルダーが残っていないか）
- [ ] **画像パス**: `<img src="...">` のパスが正しいか（大文字小文字含む）
- [ ] **コメントアウト**: 制作途中のコメントアウト・デバッグ用コードが残っていないか
- [ ] **インデント**: ファイル内でインデント（タブ／スペース）が統一されているか

### D. アクセシビリティ（基本）

- [ ] **画像の alt**: 上記「画像」セクションと連動
- [ ] **フォームのラベル**: `<label for="...">` または `aria-label` が紐付いているか
- [ ] **ボタンとリンクの使い分け**: ページ遷移は `<a>`、アクションは `<button>` を使い分けているか
- [ ] **キーボード操作**: フォーカス可能な要素にフォーカススタイル（`:focus`）が残っているか（`outline: none` で消していないか）
- [ ] **色のコントラスト**: テキストと背景のコントラスト比が WCAG AA 基準（通常 4.5:1、大きな文字 3:1）を満たしているか

### E. レスポンシブ

- [ ] **viewport メタタグ**: C セクションと連動
- [ ] **横スクロール**: モバイル幅で横スクロールが発生していないか
- [ ] **ブレークポイント**: PC / タブレット / モバイルで主要なコンテンツが崩れていないか
- [ ] **タッチターゲット**: ボタン・リンクが指で押せるサイズ（最低 44×44px 目安）になっているか
- [ ] **画像のはみ出し**: `max-width: 100%` 等で画像が親要素を超えないようになっているか
- [ ] **固定幅**: `width: 1000px` のような PC 想定の固定幅がモバイル時に維持されていないか

### F. SEO（基本）

- [ ] **`<title>` / description**: C セクションと連動
- [ ] **OGP タグ**: `og:title`, `og:description`, `og:image`, `og:url`, `og:type` が設定されているか
- [ ] **Twitter Card**: `twitter:card` 等が設定されているか（必要に応じて）
- [ ] **構造化データ**: 必要なら JSON-LD で組織・パンくず等の構造化データを記述
- [ ] **canonical**: 重複コンテンツの恐れがあるページに `<link rel="canonical">` を設定
- [ ] **robots.txt / sitemap.xml**: 必要に応じて配置されているか
- [ ] **favicon**: 設定されているか（`favicon.ico`、`apple-touch-icon` 等）

### G. パフォーマンス（基本）

- [ ] **画像の遅延読み込み**: ファーストビュー外の `<img>` に `loading="lazy"` を付与
- [ ] **CSS / JS の読み込み位置**: CSS は `<head>`、JS は `</body>` 直前または `defer` / `async` 指定
- [ ] **使っていない外部リソース**: 開発中に入れたが不要になった CDN / フォント / ライブラリが残っていないか
- [ ] **コンソールエラー**: ブラウザのコンソールにエラー・警告が出ていないか

### H. リンク・最終確認

- [ ] **すべての内部リンク**: ページ内の全リンクが意図したページへ遷移するか（実際にクリックして確認）
- [ ] **外部リンク**: `target="_blank"` の場合は `rel="noopener"` を付与
- [ ] **メール・電話**: `mailto:`, `tel:` のリンクが正しいか
- [ ] **フォーム送信**: フォームがある場合、送信先・送信後の挙動が正しいか
- [ ] **404 ページ**: カスタム 404 ページが用意されているか（必要に応じて）

---

## 報告時のスタンス

- ✅ / ⚠️ / ⏭️ の数を冒頭にサマリーとして書く（例: 「✅ 32項目 / ⚠️ 3項目 / ⏭️ 5項目」）
- ⚠️ 要修正は**重要度の高い順**に並べる（リンク切れ・画像未表示・コンソールエラー等は最上位）
- スキップした項目は理由を必ず添える（「該当なし」「フォームなし」「ユーザー指示で対象外」等）
- このチェックは **AI 単独で完結しない**。表示崩れやコントラストなどは目視確認が必要なので、その旨を報告に書く
