# リサーチ部 — 部門別スキルシート（Part 2）

## 部門概要

技術トレンド調査、競合分析、市場規模・成長性の調査、技術選定のための比較調査、Web検索による情報収集を担い、全社の意思決定を情報面から支える。

---

## 評価レベル定義（共通）

| レベル | 名称 | 基準 |
|-------|------|------|
| Lv.1 | 学習中 | マニュアルや指導のもとで作業できる |
| Lv.2 | 実践 | 定型タスクを一人で完遂できる |
| Lv.3 | 自律 | 要件から成果物まで自律的に遂行できる |
| Lv.4 | リード | 領域をリードし、後輩育成・改善を推進できる |
| Lv.5 | エキスパート | 社外にも通用する専門性を持ち、戦略的判断をリードできる |

## 部門固有スキル（RS-01〜RS-09）

### RS-01: 情報収集・ソーシング

目的に適した情報ソース（Web検索、論文、公的統計、業界レポート等）を選択し、網羅的かつ効率的に一次情報を収集できる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | Google検索で基本的なキーワード検索ができる。指示されたURLから情報を収集し、要約できる | 検索演算子（AND/OR/site:等）の使い方を習得する。主要な情報ソース一覧を把握する |
| Lv.2 | 検索演算子や絞り込み機能を活用し、目的の情報に素早くたどり着ける。複数ソースを比較して信頼性を判断できる | 業界別の主要情報ソース（官公庁統計、業界団体レポート等）を一通り把握する。RSSやアラートで情報収集を自動化する |
| Lv.3 | テーマに応じて最適な情報ソースを自ら選定し、一次情報と二次情報を使い分けた調査設計ができる。英語の情報ソースも活用できる | 海外の主要リサーチソース（Statista、Gartner、CB Insights等）の特性を理解する。学術論文の検索手法を習得する |
| Lv.4 | 社内の調査ナレッジベースを構築・運用し、チーム全体の情報収集力を底上げできる。情報ソースの信頼性評価基準を策定できる | 社内向け情報ソースガイドを作成・更新する。新しい情報収集ツールの評価と導入を主導する |
| Lv.5 | 独自の情報ネットワーク（業界関係者、専門家等）を構築している。世界レベルのOSINT手法を駆使し、公開情報から非自明なインサイトを導出できる | OSINT関連のカンファレンスに参加する。情報収集手法について社外発信を行う |

### RS-02: 市場調査・市場分析

対象市場の規模、成長率、セグメント構造、主要プレイヤーを定量・定性の両面で調査・分析できる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 市場調査の基本用語（TAM/SAM/SOM、CAGR等）を説明できる。公開データを使って市場規模を調べられる | マーケティングリサーチの入門書を通読する。業界レポートの読み方を学ぶ |
| Lv.2 | 指定されたテーマで市場概況レポートを作成できる。公的統計や業界データを用いた市場規模の算出ができる | 矢野経済研究所・富士キメラ等の市場調査レポートの構造を把握する。Excelで基本的な市場分析を行う |
| Lv.3 | 複数データソースを組み合わせたボトムアップ/トップダウンの市場規模推計ができる。市場セグメンテーションの設計と各セグメントの分析を自力で行える | フェルミ推定の手法を体系的に学ぶ。業界カオスマップの作成を経験する |
| Lv.4 | 新規事業・新規サービスの市場機会分析をリードできる。調査結果から経営判断に資する戦略示唆を導出できる | 戦略コンサルの市場分析フレームワークを深く理解する。社内の事業計画策定に調査面から参画する |
| Lv.5 | 市場のパラダイムシフトを予見し、中長期の市場動向を独自に分析・発信できる。業界内で信頼されるリサーチャーとして認知されている | 市場分析レポートを社外に公開する。業界団体やカンファレンスでの発表を行う |

### RS-03: 競合分析

競合企業の戦略、サービス、価格体系、強み・弱みを体系的に調査し、自社との比較分析を行える。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 指示された競合企業のWebサイトや公開情報から基本情報（事業内容、従業員数、売上等）を収集できる | 競合分析テンプレートの使い方を覚える。SimilarWeb等の基本ツールを使ってみる |
| Lv.2 | 競合比較表を作成し、機能・価格・ポジショニングの違いを整理できる。WEB制作業界の主要プレイヤーを把握している | ポーターの5フォース分析を実践する。SimilarWeb、BuiltWith等のツールで定期的に競合を観察する |
| Lv.3 | 競合のビジネスモデル、収益構造、技術スタックまで踏み込んだ分析ができる。SWOT分析やポジショニングマップを自力で作成できる | 有価証券報告書・IR資料の読み方を習得する。バリューチェーン分析を実務で使う |
| Lv.4 | 競合動向のモニタリング体制を構築し、変化を即時に検知・共有できる。競合分析から自社の差別化戦略への具体的な提言ができる | 競合インテリジェンスの自動化ツールを導入・運用する。経営層への競合ブリーフィングを定例化する |
| Lv.5 | 業界全体のエコシステムを俯瞰し、競合の次の一手を予測できる。競合分析のフレームワーク自体を自社に最適化して設計できる | 競合インテリジェンスの国際的な手法（SCIP等）を学ぶ。業界横断的なベンチマーキングを実施する |

### RS-04: 技術トレンド調査

WEB・IT・AI領域の技術トレンドを継続的にウォッチし、自社事業への影響と活用可能性を評価できる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 主要な技術ニュースサイト（TechCrunch、ITmedia等）を定期的にチェックできる。話題の技術について概要を説明できる | 日次で技術ニュースを読む習慣をつける。テック系Podcastやニュースレターを購読する |
| Lv.2 | 技術トレンドを整理し、社内向けのトレンドレポートを作成できる。Hype Cycleやテクノロジーレーダーの読み方を理解している | Gartner Hype Cycle、ThoughtWorks Technology Radarを定期的に確認する。月次のトレンドサマリーを作成する |
| Lv.3 | 技術トレンドが自社事業に与える影響を評価し、機会と脅威を分析できる。PoC（概念実証）の要否判断ができる | 技術カンファレンス（Google I/O、AWS re:Invent等）の内容をフォローする。自社に関連する技術のPoCを企画・実施する |
| Lv.4 | 全社の技術ロードマップ策定に必要なトレンド情報を体系的に提供できる。技術選定の意思決定に資する比較評価をリードできる | 社内テックレーダーを作成・運用する。技術戦略会議でのインプットを定例化する |
| Lv.5 | 業界に先んじて次世代技術のインパクトを予見し、自社の技術戦略に反映できる。技術トレンド分析の第一人者として社外からも認知されている | 技術トレンドに関する社外発信（登壇・執筆）を行う。海外の先端技術コミュニティに参加する |

### RS-05: 技術比較・選定調査

フレームワーク、ツール、SaaS等の技術選定に必要な比較調査を実施し、根拠ある推奨を行える。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 指定された複数ツールの公式サイトから機能・料金を調べ、比較表にまとめられる | 比較表テンプレートの使い方を習得する。G2、Capterra等のレビューサイトを活用する |
| Lv.2 | 評価軸（機能、コスト、学習コスト、コミュニティの活発さ等）を設定し、定量的な比較ができる。ユーザーレビューやベンチマーク情報を収集・整理できる | 技術選定で使われる評価フレームワーク（加重スコアリング等）を学ぶ。GitHub Starsやnpmトレンドの読み方を理解する |
| Lv.3 | 要件に応じた評価基準を自ら設計し、総合的な推奨案を提示できる。実際にハンズオンで触って得た知見を調査に反映できる | 主要技術領域ごとの選定チェックリストを作成する。PoCの計画・実施を経験する |
| Lv.4 | 技術選定プロセス全体を設計・リードできる。選定結果の妥当性を経営・技術の両面から説明できる。技術負債のリスク評価も含めた提言ができる | ADR（Architecture Decision Records）の運用を導入する。過去の技術選定の振り返りを制度化する |
| Lv.5 | 業界標準の技術選定基準を自社コンテキストに最適化して体系化できる。ベンダーとの技術的な交渉・評価を統括できる | 技術選定に関するナレッジを社外に発信する。技術デューデリジェンスの手法を習得する |

### RS-06: データ分析・可視化

調査データを統計的に分析し、分かりやすいグラフや図表で可視化して意思決定者に伝えられる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | Excelで基本的なグラフ（棒グラフ、円グラフ、折れ線グラフ）を作成できる。平均、中央値、構成比等の基本指標を算出できる | Excel/スプレッドシートのグラフ機能を一通り使えるようにする。統計の入門書を読む |
| Lv.2 | 調査データを目的に応じた適切なグラフ形式で可視化できる。クロス集計や時系列分析などの基本的なデータ加工ができる | データビジュアライゼーションの基本原則を学ぶ。Google Data Studio等のBIツールを使ってみる |
| Lv.3 | 統計的手法（回帰分析、相関分析等）を用いたデータ分析ができる。データから有意なインサイトを抽出し、ストーリーとして構成できる | 統計学の基礎（推定・検定）を体系的に学ぶ。Pythonでのデータ分析（pandas、matplotlib）を習得する |
| Lv.4 | 社内のデータ分析基盤の設計・運用をリードできる。分析手法の標準化とチームメンバーへの教育ができる | 高度なBIツール（Tableau、Power BI等）を業務に導入する。データ分析の社内研修を企画・実施する |
| Lv.5 | 大規模データセットの高度な分析（機械学習を活用した予測分析等）ができる。データドリブンな意思決定文化の醸成を牽引できる | データサイエンス関連の資格取得を目指す。分析手法や知見を社外に発信する |

### RS-07: レポーティング・プレゼンテーション

調査結果を読み手に合わせて構造化し、説得力のあるレポートやプレゼン資料にまとめられる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 指定されたテンプレートに沿って調査結果を記入できる。事実と意見を区別して記述できる | ビジネス文書の基本ルールを学ぶ。社内の過去レポートを読み、構成パターンを理解する |
| Lv.2 | 調査結果を論理的な構成（背景→調査方法→結果→考察）でまとめたレポートを一人で作成できる。図表を効果的に配置できる | ロジカルライティングの書籍を読む。PREP法やピラミッド構造を実践する |
| Lv.3 | 読み手（経営層 / 技術者 / 営業等）に合わせてレポートの粒度と表現を最適化できる。エグゼクティブサマリーを的確に書ける | バーバラ・ミントの「考える技術・書く技術」を精読する。プレゼンテーションスキル研修を受講する |
| Lv.4 | 社内のレポートテンプレートやレポーティング基準を策定できる。メンバーのレポートをレビューし、品質を引き上げられる | レポート品質基準のチェックリストを作成する。社内レビュー会を主催する |
| Lv.5 | 調査レポートが社外のクライアントや業界レポートとしても通用するクオリティで作成できる。データストーリーテリングの手法に精通している | 社外向けホワイトペーパーや業界レポートを執筆する。プレゼンテーション関連の社外登壇を行う |

### RS-08: リサーチフレームワーク活用

PEST分析、3C分析、バリューチェーン分析等の戦略フレームワークを調査に適用できる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 代表的なフレームワーク（PEST、3C、SWOT、5フォース）の目的と構成要素を説明できる | 経営戦略の入門書を通読する。各フレームワークの事例を調べてノートにまとめる |
| Lv.2 | 指示されたフレームワークを使って実際の調査テーマを分析できる。テンプレートに沿って各要素を埋められる | 実在企業を題材にフレームワーク分析を練習する。ケーススタディ集を活用する |
| Lv.3 | 調査テーマに応じて最適なフレームワークを自ら選択し、組み合わせて分析できる。フレームワークの限界を理解し、補完的に活用できる | 応用的なフレームワーク（ブルーオーシャン戦略、ジョブ理論等）も習得する。実務で複数フレームワークの組み合わせを試す |
| Lv.4 | 自社の事業特性に合わせたカスタムフレームワークを設計できる。フレームワーク活用の社内標準を策定し、メンバーを指導できる | フレームワーク活用ガイドを作成する。社内ワークショップを企画・運営する |
| Lv.5 | フレームワークを戦略的に拡張し、業界特有の分析手法を体系化できる。外部コンサルタントと対等に議論し、分析品質を担保できる | 戦略コンサルのケースメソッドを深く研究する。独自フレームワークを社外に発信する |

### RS-09: AI活用リサーチ

ChatGPT、Claude等の生成AIや各種AIツールを調査業務に活用し、リサーチの効率と品質を向上させられる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 生成AIに基本的な質問をして情報収集の補助に使える。AIの回答には誤りが含まれる可能性があることを理解している | ChatGPT/Claudeの基本的な使い方を習得する。AIリテラシーの基礎を学ぶ |
| Lv.2 | 効果的なプロンプトを設計し、調査の下調べや情報整理にAIを活用できる。AI出力のファクトチェックを確実に行える | プロンプトエンジニアリングの基本を学ぶ。Perplexity AI等のAI検索ツールを使いこなす |
| Lv.3 | 調査ワークフローの中でAI活用ポイントを特定し、人間の判断とAIの効率を最適に組み合わせた調査設計ができる。AIによる要約・翻訳・分類を業務に組み込める | RAG（検索拡張生成）の仕組みを理解する。AIを組み込んだ調査テンプレートを作成する |

| Lv.5 | AIエージェントを活用した自動リサーチシステムを設計・運用できる。AI活用リサーチの最前線を常に把握し、社内外に発信できる | AIリサーチ自動化のプロトタイプを開発する。AI×リサーチの知見を社外カンファレンスで発表する |
