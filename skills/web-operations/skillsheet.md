### WO-01: SEO内部対策

サイト構造、メタタグ、構造化データ、内部リンク設計など、テクニカルSEOの施策を適切に実施し、検索エンジンからの評価を高める能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 指導のもとでtitle・descriptionの設定やalt属性の記述ができる | 書籍『10年つかえるSEOの基本』を読了する / Google公式SEOスターターガイドを通読する |
| Lv.2 | 既存ページのメタ情報・見出し構造の修正を一人で実施できる | ウェブ解析士の学習 / 実案件でのメタ情報一括最適化を経験する |
| Lv.3 | サイト全体のクロール・インデックス状況を分析し、技術的課題を特定して改善施策を自律的に立案・実行できる | 構造化データ（JSON-LD）の実装経験を積む / Screaming Frog等のクロールツールを習熟する |
| Lv.4 | SEO内部対策のガイドラインを策定し、チーム全体の施策品質を標準化できる。後輩への指導ができる | SEO検定1級の取得 / 社内SEOチェックリスト・ガイドラインの策定を主導する |
| Lv.5 | 検索アルゴリズムの変動を先読みし、中長期的なテクニカルSEO戦略を策定できる。業界カンファレンスで登壇できるレベルの知見を持つ | 海外SEOカンファレンス（SMX等）への参加 / 技術ブログ・登壇で社外発信する |

被リンク獲得戦略やコンテンツSEO（検索意図に沿ったコンテンツ設計）を通じて、オーガニック検索からの流入を持続的に拡大する能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | キーワードリサーチツール（ラッコキーワード、Ubersuggest等）の基本操作ができる | 書籍『沈黙のWebライティング』を読了する / キーワード調査の基本手順を学ぶ |
| Lv.2 | 指定キーワードに対する検索意図を分析し、SEO記事の構成案を作成できる | Ahrefs/SEMrushの基本操作を習得する / 月5本以上のSEO記事構成を経験する |
| Lv.3 | 競合分析に基づくコンテンツ戦略を立案し、被リンク獲得施策を含めた包括的なSEO施策を自律的に推進できる | コンテンツマーケティング関連の資格取得 / リンクビルディング施策を3件以上主導する |

### WO-05: Core Web Vitals最適化

LCP（Largest Contentful Paint）、INP（Interaction to Next Paint）、CLS（Cumulative Layout Shift）などのWebパフォーマンス指標を計測・分析し、ユーザー体験とSEO評価を改善する能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | PageSpeed InsightsやLighthouseでCWVスコアを計測し、各指標の意味を理解できる | web.dev公式ドキュメントを通読する / Lighthouse操作に習熟する |
| Lv.2 | 画像最適化（WebP変換・遅延読み込み）やCSS/JSの圧縮など、定型的なパフォーマンス改善を一人で実施できる | 書籍『Webフロントエンド ハイパフォーマンスチューニング』の関連章を学習する / 実案件で画像最適化を3件以上経験する |
| Lv.3 | CrUX（Chrome User Experience Report）のフィールドデータを分析し、ボトルネックを特定して包括的な改善計画を策定・実行できる | CrUXダッシュボードの構築経験 / CDN・キャッシュ戦略の設計経験を積む |
| Lv.4 | パフォーマンスバジェットの策定とCI/CDへの組み込みを主導し、チーム全体のパフォーマンス意識を向上できる | WebPageTest APIを活用した自動テスト環境の構築 / パフォーマンス改善ガイドラインの策定を主導する |
| Lv.5 | ブラウザレンダリングの仕組みを深く理解し、最新のWeb標準（Speculation Rules API等）を活用した先進的な最適化戦略を提案・実行できる | W3C Web Performance WGの動向をウォッチする / パフォーマンス最適化に関する技術記事を社外発信する |

### WO-06: サイト保守・障害対応

CMS・サーバー・ドメイン・SSL証明書などのインフラ保守を行い、障害発生時には迅速に原因を特定・復旧する能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 定期バックアップの実行確認やSSL証明書の有効期限チェックなど、マニュアルに沿った保守作業ができる | サーバー基礎知識（Linux基礎コマンド）を学習する / 社内保守マニュアルを通読する |
| Lv.2 | WordPress本体・プラグイン・テーマのアップデートをテスト環境で検証のうえ一人で実施できる。軽微な障害（500エラー、SSL期限切れ等）に対処できる | .htaccess・php.iniの基本設定を習得する / ステージング環境の構築・運用を経験する |
| Lv.3 | 障害発生時にサーバーログを分析して原因を特定し、復旧対応を自律的に完遂できる。保守計画を策定できる | 書籍『Webエンジニアが知っておきたいインフラの基本』を読了する / 障害対応を5件以上主担当として経験する |
| Lv.4 | 保守運用体制の設計・改善を主導し、障害対応フローの標準化とチームへの展開ができる。SLAの策定に関与できる | ITIL Foundation資格の取得 / 障害対応手順書・エスカレーションフローの整備を主導する |
| Lv.5 | マルチサイト・マルチサーバー環境の保守戦略を策定し、ゼロダウンタイムを目指した運用アーキテクチャを設計できる | AWS/GCP等のクラウドインフラ資格の取得 / DR（災害復旧）計画の策定経験を積む |
