## 4. 運用フロー

### 4-1. 導入フロー（初回）

```
Step 1: スキルシートの配布
  └─ 各社員に共通テンプレート（skillsheet-template.md）と
     所属部門のスキルシート（例: web-production/skillsheet.md）を配布

Step 2: 自己評価の実施
  └─ 各社員が共通スキル21項目 + 部門固有スキルに対して
     Lv.1〜Lv.5 の自己評価を記入

Step 3: 上長面談（1on1）
  └─ 自己評価を基に上長と面談
  └─ 上長評価を記入、認識ギャップを話し合う

Step 4: 学習計画の策定
  └─ 現状レベルと目標レベルのギャップから
     次の半期で重点的に伸ばすスキルを2〜3項目選定
  └─ 推奨学習アクションを参考に具体的な計画を立てる

Step 5: 運用開始
  └─ 学習計画に基づき日常業務で実践
```

### 4-2. 定期更新サイクル（半期ごと）

```
[半期サイクル]

月1: 自己評価の更新
  └─ 前回からの成長・変化を反映して各項目のレベルを再評価

月2: 上長面談
  └─ 上長評価とのすり合わせ
  └─ 前回の学習計画の振り返り（達成度の確認）

月3: 次期学習計画の策定
  └─ 新たな重点スキル2〜3項目を選定
  └─ 具体的な学習アクション・達成期限を設定

月4〜6: 実践期間
  └─ 学習計画に基づく実践
  └─ 必要に応じて月次の簡易振り返り
```

### 4-3. 年間スケジュール（推奨）

| 時期 | アクション |
|------|-----------|
| 4月 | 上半期の自己評価 + 上長面談 + 学習計画策定 |
| 7月 | 中間振り返り（簡易） |
| 10月 | 下半期の自己評価 + 上長面談 + 学習計画策定 |
| 1月 | 中間振り返り（簡易） |
| 3月 | 年度末の総合振り返り + 次年度の育成方針策定 |

## 5. 運用上の注意事項

1. **全スキルでLv.5を目指す必要はない** -- 個人の専門領域でLv.4〜5を目指し、隣接領域でLv.2〜3を確保する「T字型人材」が理想
2. **自己評価と上長評価のギャップは成長のヒント** -- ギャップがあること自体は問題ではなく、対話の材料として活用する
3. **スキルレベルの絶対値だけでなく成長度も重視** -- Lv.1からLv.2への成長もLv.4からLv.5への成長も等しく評価する
4. **部門固有スキルと共通スキルの棲み分け** -- 共通スキルは「全社員の基礎素養」、部門固有スキルは「専門領域の深い実践力」として運用する

## 6. 改訂・追加のルール

- 新部門の追加: `common/department-template.md` をコピーし、部門固有のスキル項目を定義する
- スキル項目の追加: 各部門7〜10項目の範囲を守る。超える場合は既存項目との統合を検討する
- レベル定義の変更: 全社に影響するため、company-vision.md の改訂と全部門への通知が必要
- 品質レビュー: 年1回、品質管理部（Kaito）がレビューを実施し、quality-review.md を更新する
