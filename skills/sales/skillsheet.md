### SL-01: 顧客ヒアリング・課題整理

顧客のビジネス背景・現状の課題・理想の状態を的確にヒアリングし、WEB制作・システム開発・AI活用など自社サービスに結びつく形で課題を構造化・整理する能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 用意されたヒアリングシートに沿って顧客に質問ができる | ヒアリングシートのテンプレートを熟読し、先輩の商談に同席して記録を取る |
| Lv.2 | 定型的なWEB制作案件において、顧客の要望を一人で聞き取り議事録にまとめることができる | 過去のヒアリング議事録を分析し、質問パターンを体系的に学ぶ |
| Lv.3 | 顧客が言語化できていない潜在課題を引き出し、課題の優先順位を整理した上で提案の方向性を自律的に設計できる | SPIN話法やコンサルティングセールスの手法を学び、ロールプレイで実践する |
| Lv.4 | 複雑な要件のヒアリング手法を標準化し、後輩のヒアリングスキルを育成できる。ヒアリングシートの改善を主導できる | ヒアリングフレームワークを設計し、社内研修プログラムを企画・実施する |
| Lv.5 | 経営層レベルの課題に対して戦略的なヒアリングを行い、DX推進や事業変革を含む大型案件の要件定義をリードできる | 経営コンサルティング手法やデザインシンキングを習得し、業界カンファレンスで事例を発表する |

### SL-02: 提案書・企画書作成

顧客の課題に対して、自社のWEB制作・システム開発・AI活用サービスを組み合わせた解決策を、論理的かつ視覚的に優れた提案書・企画書としてまとめる能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 既存のテンプレートを使い、指示された構成に沿って提案書の各セクションを埋めることができる | 過去の受注提案書を10件以上読み込み、構成パターンを学ぶ |
| Lv.2 | 定型的なWEBサイト制作・リニューアル案件の提案書を一人で作成できる | 提案書デザインの基本（レイアウト・配色・図解）を学び、PowerPoint/Canvaの操作を習熟する |
| Lv.3 | 複合サービス（WEB+システム+AI）を組み合わせた提案書を自律的に作成し、ROI試算や導入効果を数値で示すことができる | ロジカルシンキング・ストーリーテリング手法を学び、提案書のセルフレビュー基準を確立する |
| Lv.4 | 提案書の品質基準を策定し、チームメンバーの提案書をレビュー・改善指導できる。提案テンプレートの更新を主導できる | 提案書レビューチェックリストを設計し、社内ナレッジベースを構築・運用する |
| Lv.5 | 業界トレンドや先進事例を踏まえた戦略的提案書を作成し、大型案件や新規事業領域の提案をリードできる | 業界の先端事例を継続的にリサーチし、提案メソドロジーを社外セミナーで共有する |

### SL-03: 見積書作成・価格設計

サービス内容・工数・難易度を適切に評価し、利益率を確保しつつ顧客にとって納得感のある見積書を作成する能力。価格の根拠を論理的に説明できることを含む。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 既存の見積テンプレートと単価表に基づき、指示された項目の見積書を作成できる | 見積テンプレートと単価表を理解し、過去の見積事例を確認する |
| Lv.2 | 定型的なWEBサイト制作案件の見積を、工数と単価を算出して一人で作成できる | 工数見積の基本手法（類推法・ボトムアップ法）を学び、実績データと比較検証する |
| Lv.3 | 複合案件の見積を自律的に作成し、制作・開発チームと連携して工数を精査できる。顧客に価格の根拠を説明できる | 原価計算・利益率管理の知識を深め、過去案件の見積精度を分析して改善する |
| Lv.4 | 見積基準・単価テーブルの改定を主導し、チームの見積精度を向上させる仕組みを構築できる | 業界の価格動向を調査し、社内の見積プロセス改善プロジェクトをリードする |
| Lv.5 | サブスクリプション型・成果報酬型など新しい価格モデルを設計し、事業戦略に基づいた料金体系を構築できる | SaaS/サブスクリプションビジネスの料金設計を研究し、価格戦略の専門知識を深める |

### SL-04: 商談シナリオ設計・プレゼンテーション

顧客の状況・意思決定プロセスに合わせた商談の流れを設計し、自社サービスの価値を効果的に伝えるプレゼンテーションを行う能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 用意された商談資料を使い、先輩のサポートのもとで顧客への説明ができる | 先輩の商談に同席し、話の流れ・切り返し方を観察・記録する |
| Lv.2 | 定型的な案件の商談を一人で進行し、自社サービスの基本的な説明・デモができる | プレゼンテーションの基本（構成・話し方・資料の見せ方）を練習し、社内ロールプレイに参加する |
| Lv.3 | 顧客の意思決定者・影響者を見極め、それぞれに合わせた商談シナリオを自律的に設計・実行できる | セールスプロセス手法（MEDDIC、Challengerセールス等）を学び、商談振り返りを習慣化する |
| Lv.4 | 商談ノウハウを体系化し、営業チームのプレゼン力を底上げする研修・フィードバック体制を構築できる | 営業トレーニングプログラムを設計し、メンバーの商談同行・フィードバックを定期的に実施する |
| Lv.5 | 大型案件・コンペにおいて戦略的な商談をリードし、経営層への提案や複数回にわたる交渉プロセスを統括できる | トップセールスの交渉術・アカウントプランニングを研究し、社外の営業コミュニティで知見を共有する |

### SL-05: 契約書ドラフト作成・交渉

業務委託契約・NDA・SLAなどの契約書を適切にドラフトし、顧客との契約条件交渉を行う能力。リスクを理解し、自社と顧客双方にとって適正な条件を導く。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 契約書テンプレートの各条項の意味を理解し、指示に基づいて必要事項を記入できる | 契約書テンプレートの各条項を学び、基本的な法務用語を理解する |
| Lv.2 | 定型案件の業務委託契約書・NDAを一人でドラフトし、基本的な条件（納期・支払条件・瑕疵担保）を設定できる | 契約実務の基礎（民法・下請法の基本）を学び、過去の契約書パターンを整理する |
| Lv.3 | 非定型の契約条件（成果報酬・知的財産権・SLA）を自律的に設計し、顧客との条件交渉ができる | 契約交渉のケーススタディを分析し、リスク管理の観点を強化する |
| Lv.4 | 契約書テンプレートの更新・改善を主導し、チームメンバーの契約実務をレビュー・指導できる。リスクの高い条項を事前に識別できる | 法務セミナーに参加し、業界の契約トレンド（SaaS契約・アジャイル契約等）を研究する |
| Lv.5 | 新しい取引形態に対応した契約スキームを設計し、顧問弁護士と連携して法的リスクを最小化した契約体系を構築できる | IT契約に関する専門的な法務知識を深め、業界団体の契約ガイドライン策定に参画する |

### SL-06: 顧客関係管理（CRM）

顧客情報を体系的に管理し、適切なタイミングでフォローアップを行い、継続的な取引関係と顧客満足度の向上を実現する能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | CRMツール（スプレッドシート等）に商談情報・顧客情報を正確に記録できる | CRMツールの基本操作を習得し、データ入力のルールを理解する |
| Lv.2 | 担当顧客のフォローアップを定期的に実施し、案件の進捗状況を適切に管理・報告できる | 顧客管理の基本フレームワーク（顧客ライフサイクル）を学び、フォロー計画を立てる |
| Lv.3 | 顧客データを分析して優先順位を設定し、顧客満足度を高めるための施策を自律的に企画・実行できる | CRM分析手法（RFM分析・顧客セグメンテーション）を学び、データドリブンな顧客管理を実践する |
| Lv.4 | CRM運用ルールを策定し、チーム全体の顧客管理品質を向上させる仕組みを構築できる。顧客満足度の計測・改善サイクルを主導できる | CRMシステムの導入・カスタマイズを検討し、カスタマーサクセスの方法論を研究する |
| Lv.5 | 顧客LTV最大化のための戦略を設計し、全社的な顧客エンゲージメント戦略をリードできる | カスタマーサクセス戦略・ABM（アカウントベースドマーケティング）を研究し、業界のベストプラクティスを確立する |

### SL-07: クロスセル・アップセル提案

既存顧客の利用状況・ビジネス課題を把握し、WEB運用・システム追加開発・AI導入など自社の他サービスを適切なタイミングで提案して取引を拡大する能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 自社の全サービスラインナップを理解し、既存顧客に基本的なサービス紹介ができる | 自社サービスカタログを熟読し、各部門の対応領域・事例を把握する |
| Lv.2 | 既存顧客の契約内容を把握し、関連サービスの提案タイミングを見極めて定型的な追加提案ができる | クロスセル・アップセルの基本手法を学び、成功事例を分析する |
| Lv.3 | 顧客のビジネス課題を先読みし、WEB+システム+AIを組み合わせた包括的な追加提案を自律的に設計・実行できる | アカウントプランニング手法を学び、顧客ごとの成長シナリオを設計する |
| Lv.4 | 既存顧客の取引拡大戦略を策定し、チームの追加提案活動を体系化・推進できる。クロスセル成功率の改善を主導できる | アカウントマネジメントのベストプラクティスを研究し、提案パターンをナレッジ化する |
| Lv.5 | 顧客のDXパートナーとしてのポジションを確立し、年間取引額の大幅拡大を実現する戦略的アカウント経営ができる | 戦略的パートナーシップの構築手法を研究し、業界内でのリレーション構築をリードする |

### SL-08: 競合分析・差別化戦略

競合他社のサービス内容・価格・強み・弱みを分析し、自社のWEB制作・システム開発・AI活用の強みを活かした差別化ポイントを明確にして営業活動に反映する能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 主要競合他社のサービス内容・価格帯をリストアップし、基本的な比較表を作成できる | 競合他社のWEBサイト・事例を調査し、サービス比較資料を読み込む |
| Lv.2 | 商談で競合が出てきた際に、自社の強みと競合との違いを顧客に説明できる | SWOT分析・3C分析の基本を学び、自社と競合の比較分析を実践する |
| Lv.3 | 市場動向と競合の戦略を分析し、案件ごとに最適な差別化ポイントを設計して提案に反映できる | 競合インテリジェンスの手法を学び、定期的な市場・競合レポートを作成する |
| Lv.4 | 競合分析の仕組みを構築し、チーム全体の競合対策力を強化する戦略を策定・推進できる | 競合分析フレームワークを設計し、営業チームへの定期的な競合情報共有会を主催する |
| Lv.5 | 業界全体のポジショニングを見据えた差別化戦略を経営陣に提言し、自社のブランド価値向上をリードできる | ポジショニング戦略・ブルーオーシャン戦略を研究し、業界セミナーで差別化戦略を発信する |

### SL-09: 社内連携・要件伝達

営業段階で把握した顧客要件・期待値を、WEB制作部・システム開発部・AI活用推進部などの制作・開発チームに正確かつ過不足なく伝達し、プロジェクト成功に導く能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 商談で確認した顧客要件を、所定のフォーマットに沿って社内共有資料にまとめることができる | 社内の要件伝達フォーマット・ワークフローを理解し、過去の引き継ぎ資料を確認する |
| Lv.2 | 制作・開発チームへのキックオフミーティングを一人で実施し、基本的な要件・スケジュール・予算を正確に伝達できる | プロジェクト管理の基礎（スコープ・WBS）を学び、要件定義書の書き方を習得する |
| Lv.3 | 技術的な実現可能性を事前に制作・開発チームと擦り合わせ、顧客期待と技術制約のギャップを調整した上で要件を確定できる | 技術知識（WEB・システム・AI）の基礎を広く学び、部門横断プロジェクトの経験を積む |
| Lv.4 | 営業-制作間の連携プロセスを改善し、認識齟齬による手戻りを組織的に削減する仕組みを構築できる | 部門間連携のベストプラクティスを研究し、連携プロセスの標準化・改善をリードする |
| Lv.5 | 全社的な顧客要件管理の仕組みを設計し、営業から納品までの一気通貫の品質保証体制をリードできる | アジャイル・リーン手法を深く理解し、組織横断のプロセス改革を推進する |
