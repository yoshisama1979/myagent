# 株式会社はなさか 求める人材像とスキル評価基準

## 目的

本ドキュメントは、株式会社はなさかが全社員に共通して求める人材像を定義し、スキルシート（自己学習ガイド兼人事評価基準）の土台とするものです。各部門の個別スキルシートは、本ドキュメントの共通基準をベースに、部門固有のスキルを追加して構成します。

---

## 1. 求める人材像

### 1-1. 基本方針

株式会社はなさかは「WEB制作・WEB運用・システム開発・AI活用」を一気通貫で提供する企業です。この事業特性から、以下の3つの資質を持つ人材を求めます。

- **越境できる人** — 自分の専門領域に閉じず、隣接領域にも関心を持ち、部門を横断して協働できる
- **自走できる人** — 指示を待つのではなく、目的を理解し、自ら課題を発見して動ける
- **学び続ける人** — 技術トレンドの変化が速い業界において、継続的に学習し、スキルを更新し続ける

### 1-2. 行動指針（バリュー）

| バリュー | 意味 |
|---------|------|
| 一気通貫 | 制作・運用・開発・AIを分断せず、顧客の課題をワンストップで解決する視点を持つ |
| 顧客起点 | 技術ありきではなく、顧客のビジネス成果から逆算して最適な手段を選ぶ |
| 構造化思考 | 曖昧な課題を整理し、論理的に分解して解決策を導く |
| オープンな共有 | 知識やノウハウを個人に閉じず、チームに還元する |

---

## 2. スキルカテゴリ体系

全社員のスキルを以下の3カテゴリに分類します。各カテゴリの中に具体的なスキル項目を設けます。

### カテゴリ一覧

| カテゴリ | 概要 | 評価ウェイトの目安 |
|---------|------|------------------|
| テクニカルスキル | 業務遂行に必要な技術的知識・実践力 | 部門により異なる（40〜60%） |
| ビジネススキル | 顧客対応、プロジェクト推進、経営理解に関する能力 | 全部門共通（20〜30%） |
| ヒューマンスキル | コミュニケーション、自己管理、チーム協働に関する能力 | 全部門共通（20〜30%） |

## 3. 共通スキル項目一覧

### 3-1. テクニカルスキル（全社共通）

全部門の社員が最低限持つべき技術的素養です。部門固有の専門スキルは各部門のスキルシートで別途定義します。

| # | スキル項目 | 概要 |
|---|-----------|------|
| T-01 | IT基礎知識 | ネットワーク、サーバー、データベースの基本概念を理解している |
| T-02 | WEB技術の基礎 | HTML/CSS/JavaScriptの基本、HTTPの仕組み、ブラウザの動作原理を理解している |
| T-03 | セキュリティ基礎 | 情報セキュリティの基本原則（機密性・完全性・可用性）を理解し、日常業務で実践できる |
| T-04 | バージョン管理（Git） | Gitの基本操作（commit, push, pull, branch）を理解し、チーム開発に参加できる |
| T-05 | AI活用リテラシー | 生成AI（LLM）の基本概念を理解し、業務効率化にAIツールを活用できる |
| T-06 | ドキュメント作成 | Markdown、スプレッドシート等を用いて、構造化された分かりやすいドキュメントを作成できる |
| T-07 | ツール活用力 | 業務で使用するツール（プロジェクト管理、チャット、CMS等）を適切に操作できる |

### 3-2. ビジネススキル

| # | スキル項目 | 概要 |
|---|-----------|------|
| B-01 | 顧客理解 | 顧客のビジネスモデル・業界・課題を把握し、提案に反映できる |
| B-02 | 要件整理 | 曖昧な依頼を構造化し、実行可能なタスクに分解できる |
| B-03 | プロジェクト管理基礎 | スケジュール、タスク、リスクの管理手法を理解し、自分の担当範囲で実践できる |
| B-04 | 見積・工数感覚 | 自分の担当作業にかかる工数を合理的に見積もれる |
| B-05 | 品質管理意識 | 成果物の品質基準を理解し、セルフチェックを習慣化できる |
| B-06 | 事業理解 | はなさかの事業領域（制作・運用・開発・AI）と、それぞれの関係性を理解している |
| B-07 | 提案力 | 課題に対して代替案を複数提示し、メリット・デメリットを整理して伝えられる |

### 3-3. ヒューマンスキル

| # | スキル項目 | 概要 |
|---|-----------|------|

## 4. 5段階評価レベルの定義

### 4-1. レベル一覧

| レベル | 名称 | 基準 | 自己学習ガイドとしての目安 |
|-------|------|------|-------------------------|
| Lv.1 | 学習中 | マニュアルや指導のもとで作業できる。基礎知識を学習中 | 入門書を読んだ・研修を受けた段階 |
| Lv.2 | 実践 | 定型的なタスクを一人で完遂できる。品質チェックは必要 | チュートリアルを完了し、簡単な実践を積んだ段階 |
| Lv.3 | 自律 | 要件から成果物まで自律的に遂行できる。非定型業務にも対応可能 | 実務経験1〜2年相当。日常業務で自走できる段階 |
| Lv.4 | リード | 該当領域をリードし、後輩育成やプロセス改善を推進できる | 実務経験3年以上相当。チームの中核として動ける段階 |
| Lv.5 | エキスパート | 社外にも通用する専門性。技術選定や戦略的判断をリードする | 業界水準で高い専門性を持ち、戦略的判断に関与できる段階 |

### 4-2. レベル運用の方針

- **全スキルでLv.5を目指す必要はない** — 個人の専門領域でLv.4〜5を目指し、隣接領域でLv.2〜3を確保する「T字型人材」を理想とする
- **自己評価 + 上長評価の二軸** — 自己学習ガイドとして自己評価を行い、人事評価時に上長が確認・調整する
- **半期ごとの更新** — 半年に1回、スキルシートを更新し、成長を可視化する
- **目標設定と連動** — 次の半期で重点的に伸ばすスキルを2〜3項目選び、学習計画を立てる

## 5. 部門別スキルシートとの関係

本ドキュメント（共通基準）と各部門のスキルシートの関係は以下の通りです。

```
company-vision.md（本ドキュメント）
├── 共通テクニカルスキル（T-01〜T-07）
├── 共通ビジネススキル（B-01〜B-07）
└── 共通ヒューマンスキル（H-01〜H-07）

各部門スキルシート（例: skills/web-production/skillsheet.md）
├── 共通スキル（本ドキュメントから継承）
└── 部門固有スキル（例: デザインスキル、フロントエンド実装スキル等）
```

### 部門固有スキルの例（今後各部門で定義）

| 部門 | 固有スキルの方向性 |
|------|------------------|
| WEB制作部 | UI/UXデザイン、フロントエンド実装、WordPress構築 |
| WEB運用部 | SEO、アクセス解析、コンテンツ運用、サイト保守 |
| システム開発部 | バックエンド開発、DB設計、API設計、テスト設計 |
| AI活用推進部 | プロンプトエンジニアリング、LLM活用設計、自動化構築 |
| 営業・顧客対応部 | 商談スキル、提案書作成、顧客関係管理 |
| マーケティング部 | コンテンツ企画、SNS運用、広告運用、データ分析 |
| 品質管理部 | テスト設計、コードレビュー、品質メトリクス管理 |
| 経営管理部 | 経理実務、契約管理、労務管理 |
| リサーチ部 | 市場調査、競合分析、トレンド分析 |
| プロジェクト管理部 | 進捗管理、リスク管理、ステークホルダー調整 |
| 経営企画部 | 事業計画策定、KPI設計、経営分析 |

## 6. 自己学習ガイドとしての活用方法

1. **現状把握** — 各スキル項目に対して自己評価（Lv.1〜5）を記入する
2. **ギャップ分析** — 求められるレベルと現状のギャップを特定する
3. **学習計画** — ギャップが大きい項目から優先的に学習計画を立てる
4. **実践と振り返り** — 業務の中で意識的にスキルを使い、半期ごとに振り返る
5. **フィードバック** — 上長や同僚からのフィードバックを受け、自己評価を修正する

---

## 7. 人事評価への転用方針

- スキルシートの評価は人事評価の **一部** として使用する（スキルだけでなく、成果・行動も評価対象）
- スキルレベルの向上そのものを評価する（絶対値だけでなく成長度も重視）
- 部門・役職ごとに「期待レベル」を設定し、到達度を評価に反映する
- 評価の透明性を担保するため、基準は全社員に公開する
