### SD-01: システムアーキテクチャ設計

システム全体の構成を俯瞰し、技術選定・コンポーネント分割・拡張性・可用性を考慮したアーキテクチャを設計できる能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 既存アーキテクチャの構成図を読み、各コンポーネントの役割を説明できる | アーキテクチャパターン入門書の学習、社内システム構成図の読解演習 |
| Lv.2 | 指定されたパターンに従い、小規模システムの構成を設計できる | MVC・レイヤード等の基本パターンを用いた設計演習、既存プロジェクトの設計レビュー参加 |
| Lv.3 | 要件に応じて適切なアーキテクチャパターンを選定し、中規模システムの設計を自律的に完遂できる | マイクロサービス・イベント駆動等の実践、設計ドキュメント作成の主担当経験 |
| Lv.4 | 複数プロジェクトのアーキテクチャをレビューし、技術的負債の解消計画を策定・推進できる | 社内アーキテクチャレビュー会の主催、技術選定ガイドラインの策定 |
| Lv.5 | 事業戦略を踏まえた全社横断のシステムアーキテクチャ戦略を策定し、技術的意思決定をリードできる | 外部カンファレンスでの登壇・執筆、業界標準アーキテクチャの調査・提言 |

### SD-02: API設計・開発（REST/GraphQL）

RESTful API / GraphQL APIの設計原則を理解し、外部・内部連携のためのAPIを設計・実装・ドキュメント化できる能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | APIの基本概念（HTTPメソッド、ステータスコード、JSON）を理解し、既存APIの仕様書を読んで利用できる | REST API入門学習、Postman等のツールを使ったAPI呼び出し演習 |
| Lv.2 | 設計ガイドラインに従い、CRUD操作のRESTful APIエンドポイントを実装できる | OpenAPI仕様の学習、既存APIへのエンドポイント追加の実務経験 |
| Lv.3 | 認証・認可・バージョニング・エラーハンドリングを含むAPI全体を自律的に設計・実装できる | OAuth2.0/JWT実装、GraphQL導入、API設計のベストプラクティス学習 |
| Lv.4 | API設計ガイドラインを策定し、チームのAPI品質をレビュー・改善できる | API設計レビュー基準の策定、後輩へのAPI設計指導、社内勉強会の開催 |
| Lv.5 | マイクロサービス間通信やサードパーティ連携を含む大規模APIエコシステムを戦略的に設計できる | gRPC/イベント駆動等の高度な通信パターン研究、API戦略のロードマップ策定 |

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | SQLの基本操作（SELECT/INSERT/UPDATE/DELETE）ができ、ER図を読んでテーブル構造を理解できる | SQL基礎学習、ER図の読解演習、既存DBスキーマの調査 |

### SD-09: DevOps・CI/CD

開発からデプロイまでのプロセスを自動化し、継続的インテグレーション・継続的デリバリーの環境を設計・構築・運用できる能力。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | CI/CDの基本概念を理解し、既存のパイプラインを利用してビルド・デプロイを実行できる | CI/CD入門学習、GitHub Actions/GitLab CIの基本操作、Dockerの基礎理解 |
