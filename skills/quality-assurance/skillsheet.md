### QA-01: コードレビュー

フロントエンド（HTML/CSS/JavaScript）およびバックエンド（PHP/Python等）のソースコードを読み、可読性・保守性・正確性の観点から品質を評価し、改善提案ができる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | レビューチェックリストに沿って、コーディング規約違反やタイポを指摘できる | コーディング規約を通読する。先輩のレビューコメントを読み、観点を学ぶ |
| Lv.2 | ロジックの誤りやエッジケースの考慮漏れを自力で発見し、具体的な修正案を提示できる | 「リーダブルコード」等のコード品質に関する書籍を読む。日常的にプルリクエストレビューに参加する |
| Lv.3 | 設計パターンやアーキテクチャの観点からコードの構造的問題を指摘し、リファクタリング案を提案できる | デザインパターン・SOLID原則を学ぶ。フロントエンド・バックエンド両方のコードベースに精通する |
| Lv.4 | チームのレビュー基準・チェックリストを策定し、レビュー文化を醸成できる。後輩のレビュースキル向上を指導できる | レビュープロセスのベストプラクティスを研究する。社内レビューガイドラインを策定・改訂する |
| Lv.5 | 静的解析ツール導入やレビュー自動化の戦略を策定できる。技術的負債の可視化と解消計画をリードできる | SonarQube等の静的解析ツールを深く理解する。社外カンファレンスでレビュー手法を発表する |

### QA-02: テスト設計

要件定義書・仕様書からテスト観点を抽出し、テスト計画を立案し、網羅性の高いテストケースを設計できる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | テストケースのフォーマットを理解し、指示されたテスト観点に基づいてテストケースを記述できる | JSTQB Foundation Levelの学習を開始する。テストケース作成の基本書を読む |
| Lv.2 | 同値分割・境界値分析・デシジョンテーブル等の技法を用いて、定型的なテストケースを一人で設計できる | JSTQB Foundation Level資格を取得する。実務プロジェクトでテスト設計を経験する |
| Lv.3 | 要件からテスト観点を自律的に抽出し、テスト計画（スコープ・スケジュール・リスク分析）を立案できる | JSTQB Advanced Level（テストアナリスト）の学習を進める。状態遷移テスト・ペアワイズテスト等の技法を実践する |
| Lv.4 | プロジェクト横断でテスト戦略を策定し、テスト設計の標準化・テンプレート整備をリードできる。後輩のテスト設計をレビュー・指導できる | テストマネジメントの書籍を読む。社内のテスト設計標準を策定する |
| Lv.5 | リスクベーステストやモデルベーステスト等の高度なテスト設計手法を導入・推進できる。テスト設計の業界動向を把握し、社外に発信できる | JSTQB Advanced Level資格を取得する。テスト関連カンファレンスでの登壇・執筆を行う |

### QA-03: テスト実行・管理

テストケースに従ったテスト実行、不具合の報告・追跡、テスト結果の集計・報告ができる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | テストケースに従って手動テストを実行し、結果を記録できる。不具合発見時にスクリーンショット付きで報告できる | テスト実行の手順書を習熟する。バグトラッキングツール（Backlog/GitHub Issues等）の基本操作を覚える |
| Lv.2 | 再現手順を明確にした不具合報告ができる。テスト進捗を集計し、定型レポートを作成できる | 不具合報告のベストプラクティスを学ぶ。テスト管理ツールの操作に習熟する |
| Lv.3 | テスト実行計画を自律的に管理し、リスクに応じた優先順位付けができる。テスト完了判定の根拠を論理的に説明できる | テストマネジメントの基礎を学ぶ。テストメトリクス（消化率・不具合検出率等）の活用方法を理解する |
| Lv.4 | 複数プロジェクトのテスト進捗を統括し、品質状況をステークホルダーに報告・説明できる。テスト実行プロセスの改善をリードできる | テスト管理のフレームワークを研究する。チーム全体のテスト実行効率改善に取り組む |
| Lv.5 | 全社的なテスト管理基盤の構築・最適化を戦略的にリードできる。テスト結果から経営判断に資する品質レポートを作成できる | テスト管理の最新トレンド（AI活用等）を調査・導入する。品質管理の外部セミナー講師を担当する |

### QA-05: セキュリティテスト

WEBアプリケーションの脆弱性診断を実施し、セキュリティリスクを評価・報告できる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | OWASP Top 10の各脆弱性の概要を説明できる。セキュリティチェックリストに沿って基本的な確認ができる | OWASP Top 10を通読する。WEB脆弱性の入門書を読む |
| Lv.2 | XSS・SQLインジェクション・CSRF等の代表的な脆弱性を手動で検証できる。脆弱性診断ツール（OWASP ZAP等）を基本操作できる | OWASP ZAPのハンズオンを実施する。DVWA等の学習環境で脆弱性検証を練習する |
| Lv.3 | 脆弱性診断の計画を立案し、自動・手動を組み合わせた診断を自律的に実施できる。発見した脆弱性の深刻度を評価し、対策案を提示できる | 情報セキュリティマネジメント試験に合格する。Burp Suite等のプロフェッショナルツールを使いこなす |
| Lv.4 | セキュリティテストの標準プロセスを策定し、チームに展開できる。インシデント対応のフローを整備し、訓練をリードできる | 情報処理安全確保支援士の学習を進める。社内セキュリティ研修を企画・実施する |
| Lv.5 | 高度な攻撃手法（APIセキュリティ、認証・認可の脆弱性等）を検証できる。全社セキュリティテスト戦略を策定し、外部監査対応を統括できる | GWAPT/OSCP等の国際資格を取得する。セキュリティコミュニティで発表・執筆する |

### QA-06: パフォーマンステスト

WEBサイト・WEBアプリケーションの表示速度・応答速度・負荷耐性を計測・評価し、改善提案ができる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | Core Web Vitals（LCP/FID/CLS）の意味を説明できる。Lighthouse等のツールで基本的な計測ができる | Core Web Vitalsの公式ドキュメントを読む。Lighthouseの使い方を習得する |
| Lv.2 | 計測結果を読み解き、ボトルネック箇所を特定できる。PageSpeed Insights・GTmetrix等の複数ツールを使い分けられる | WEBパフォーマンスの基礎書籍を読む。実際のプロジェクトで計測レポートを作成する |
| Lv.3 | 負荷テスト（JMeter/k6等）を計画・実施し、パフォーマンス要件の充足を検証できる。改善施策の優先順位を提案できる | 負荷テストツールのハンズオンを実施する。CDN・キャッシュ等のパフォーマンス最適化手法を学ぶ |
| Lv.4 | パフォーマンスバジェットを策定し、継続的なパフォーマンス監視の仕組みを構築できる。チームにパフォーマンス改善手法を指導できる | APMツール（New Relic/Datadog等）の活用方法を学ぶ。パフォーマンスチューニングの事例を研究する |
| Lv.5 | 全社的なパフォーマンス基準を策定し、アーキテクチャレベルの改善提案ができる。大規模サイトのパフォーマンス戦略を設計できる | WEB Performance Working Groupの仕様動向を追う。パフォーマンス関連の外部登壇・執筆を行う |

|-------|---------|------------------|
| Lv.1 | アクセシビリティの基本概念と重要性を説明できる。alt属性・見出し構造等の基本的なチェック項目を確認できる | WCAG 2.1の概要を読む。アクセシビリティの入門記事・動画で学ぶ |
| Lv.2 | axe DevTools・WAVE等のツールを使って自動検証ができる。WCAG 2.1 レベルAの適合基準に沿ったチェックができる | axe DevToolsの操作に習熟する。スクリーンリーダー（NVDA等）を実際に使ってみる |

| Lv.4 | アクセシビリティガイドラインを策定し、チームへの啓発・教育をリードできる。デザイン・実装段階からアクセシビリティを考慮した提案ができる | アクセシビリティ関連の勉強会を主催する。JIS X 8341-3の試験方法を理解する |

### QA-09: 品質メトリクス管理

不具合密度・テストカバレッジ・修正リードタイム等の品質指標を定義・計測・分析し、品質改善に活かせる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 不具合件数・テスト消化率等の基本的な品質指標の意味を説明できる | ソフトウェアメトリクスの入門記事を読む。プロジェクトの品質レポートを読み解く |
| Lv.2 | 定義されたメトリクスを正確に計測・集計し、定型レポートを作成できる | Excelやスプレッドシートでメトリクスダッシュボードを作成する。SPC（統計的プロセス管理）の基礎を学ぶ |
| Lv.3 | メトリクスの推移を分析し、品質リスクの早期検知や改善ポイントの特定ができる。プロジェクトに適したメトリクスを選定できる | 定量的品質管理の書籍を読む。データ分析・可視化ツール（Redash/Metabase等）を活用する |
| Lv.4 | 品質メトリクスダッシュボードを設計・構築し、組織的な品質可視化をリードできる。メトリクスに基づく改善活動をチームで推進できる | GQM（Goal-Question-Metric）手法を深く理解する。品質管理の社内研修を企画する |
| Lv.5 | 全社的な品質KPIを策定し、経営層への品質報告体制を構築できる。業界ベンチマークとの比較分析ができる | CMMIやISO 25010等の品質モデルを研究する。品質管理の外部セミナーで発表する |

### QA-10: 品質プロセス改善

QAプロセス全体の設計・運用・改善を行い、組織的な品質向上を推進できる。

| レベル | 行動指標 | 推奨学習アクション |
|-------|---------|------------------|
| Lv.1 | 現在のQAプロセス（テスト工程・レビュー工程等）の流れを説明できる | 社内のQAプロセスドキュメントを通読する。ソフトウェアテストの基礎書籍を読む |
