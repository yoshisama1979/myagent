## 2026-05-08

### 経営方針 — システム制作・運用をメイン事業へ転換

| 項目 | 内容 |
|------|------|
| カテゴリ | 経営戦略 |
| 起案者 | 社長（株式会社はなさか 代表取締役） |
| ステータス | **方針決定（実行フェーズへ移行）** |

**会社の現状認識**

- もともと **ホームページ制作会社** として設立、現在も継続
- 社長は **元システムエンジニア** のバックグラウンドあり
- 既に「少々のシステム構築」は対応可能

**戦略転換の背景**

- **AI の登場でシステム開発の障壁が大きく低下**
- 社長の元エンジニア知識を組み合わせれば、**早く・品質高く** システムを作れる
- HP制作と比べたシステム開発のメリット:
  - **単価が高い**
  - 仕様確定後はクライアントとのやり取り（ボトルネック）が **少ない**
  - 少ない作業量で **大きな成果** をクライアントに提供できる
  - **運用管理で継続収益（ストックビジネス）** が見込める

**今後の営業の流れ**

```
業務改善システム構築（フロー収益）
       ↓
運用管理契約（月額 50,000円目標 / ストック収益）
```

**事業ごとの位置付け（確定）**

| 事業 | 位置付け |
|------|---------|
| HP制作 | **顧客接点として残す**（注力分野ではない・営業の入口として活用） |
| システム制作 | **メイン**（業務改善システム + AIをふんだんに活用） |
| 運用管理 | **メイン（ストック収益）**（月額目標 50,000円） |

**狙うシステム像（確定）**

- 中心は **業務改善システム** — 会社の業務効率化のためのシステム開発・カスタマイズ
- **AI を大いに活用** した、総合的に便利な仕組みにする
- 単機能ではなく「業務全体を効率化する」総合提案を志向

**運用管理サービスの構成（確定）**

| 要素 | 内容 |
|------|------|
| サーバー構成 | **ステージング + 本番（VPS×2台）** ※サーバー費用込み |
| バックアップ | 自動化 |
| 含まれる業務 | システム保守 / バックアップ / サーバー管理 |
| 月額目標 | **50,000円程度** |
| 価格妥当性 | 業務改善システム保守の相場（5万〜20万円）の中で **やや控えめ寄り**（提供価値の割に高くない） |

**追加検討中の含有項目（要意思決定）**

- 軽微改修対応（月◯時間まで）
- 死活監視・異常通知
- SSL証明書・ドメイン更新
- 月次稼働レポート
- セキュリティパッチ適用
- ユーザー操作サポート（軽微）
- 月次定例MTG（30分）

→ 上記の確定後、**「サービス定義書」** として 1 枚にまとめてクライアント提示用テンプレ化する

**含意（他施策への波及）**

- **スキルシートで重視すべき技能**: システム開発スキル / AI活用 / インフラ運用（ステージング・本番分離、自動バックアップ）
- 提案書・営業資料・自社サイトの訴求軸を「HP制作会社」から **「業務効率化のシステム会社」** にシフト
- 提案ひな形に「**初期構築費 + 月額運用費（50,000円〜）**」のセット提案を組み込む
- 横展開のための **テンプレート化・自社プロダクト化** が重要になる可能性

**確認事項（決定済み・未決の整理）**

| # | 内容 | 状態 |
|---|---|---|
| 1 | HP事業の位置付け | ✅ 顧客接点として残す（注力外） |
| 2 | 運用管理の料金モデル | ✅ 月額50,000円目標 |
| 3 | 狙うシステム領域 | ✅ 業務改善 + AI活用 |
| 4 | 横展開戦略 | ✅ **雛形ベース + カスタマイズ重視** + **基盤ポリシー策定** |

## 今後決めておきたい項目（意思決定待ち）

### 【A】価格・サービス設計

#### A-1. 月額50,000円に含むサービス範囲の確定
> 「何を含めて、何を別料金にするか」の境界線を明文化する。曖昧だと工数が膨張するリスクあり。

含めるかどうかを **1つずつ判断したい候補**:

| 候補 | 含める / 別料金 / 採用しない |
|---|---|
| 軽微改修対応（月◯時間まで） | 未決 |
| 死活監視・異常通知（営業時間内 / 24h） | 未決 |
| SSL証明書・ドメイン更新（実費別 / 込み） | 未決 |
| 月次稼働レポート | 未決 |
| セキュリティパッチ適用 | 未決（含めるべき） |
| ユーザー操作サポート（軽微） | 未決 |
| 月次定例MTG（30分） | 未決 |

→ 確定後、**「サービス定義書」テンプレート** を作成し、提案・契約時に使う

#### A-2. 初期構築費の想定レンジ
- 小規模 / 中規模 / 大規模 ごとの目安価格
- 工数換算 + 利益率の基準
- 提案テンプレに「初期費用 + 月額50,000円〜」をセットする際の数字

### 【B】営業・対象設計

#### B-1. 営業ターゲット
- 既存HP顧客へのアップセル
- 新規開拓
- 両方（の比率・優先順位）

#### B-2. ターゲット業種・規模
- 中小企業全般 / 特定業種にフォーカス
- 従業員規模の目安

### 【C】開発・運用基盤

#### C-1. 「はなさか 業務改善システム構築ポリシー」の策定
> 雛形で固定する領域とカスタムする領域を分類するドキュメント。社員教育（スキルシート連動）・営業資料・見積基準として使う。

スコープ:
- 基盤技術（PHP・DB・フレームワーク選定）
- 認証・権限
- 共通UI
- ログ・監査
- AI連携
- インフラ（VPS・バックアップ・デプロイ）
- 業務ロジック
- 画面・帳票

→ 私（AI）の方で叩き台ドラフト作成可能。着手指示があれば作業開始

### 【D】体制・キャパシティ

#### D-1. 3名体制での同時並行キャパ
- 同時に走らせる案件数の上限想定
- 1案件あたりの平均工数想定（フェーズ別）
- これが営業ペースの上限を決める

### 社員教育用スキルシートの作成・運用ロードマップ

| 項目 | 内容 |
|------|------|
| カテゴリ | 経営／人材教育 |
| 起案者 | 社長（株式会社はなさか 代表取締役） |
| 対象 | 社員3名（社長を含む小規模体制） |
| ゴール | スキルシートを作成し、社員教育に運用する |
| ステータス | **構想段階** |

**目的**

- 社員のスキルレベルを可視化し、教育・育成・評価の基盤にする
- 自己学習の指針 + 人事評価基準として機能させる

**進め方の方針（確定）**

- **段階的アプローチ**で進める
- **まず「汎用スキル版」を作成** → その後、領域別に展開していく
- 社長が **過去に作成したスキルシート資料** をベースにする予定（後ほど共有予定）

**段階構成（仮案）**

| ステージ | 内容 | 状態 |
|---|---|---|
| **ステージ1: 汎用スキル版** | PC操作・作業スピード・基本ビジネススキル等、全社員共通の前提スキル | **着手予定**（資料受領待ち） |
| ステージ2: 制作領域版 | HTML/CSS/JS、WordPress、デザイン等 | 未定 |
| ステージ3: 開発領域版 | PHP、API、DB等 | 未定 |
| ステージ4: AI活用版 | プロンプト設計、LLM連携等 | 未定 |
| ステージ5: 営業・経営支援版 | 営業対応、提案資料、経営管理等 | 未定 |

※ ステージ2以降は、ステージ1完了後に優先順位を再検討

**ステージ1（汎用スキル版）の進行ステップ**

1. **社長から過去のスキルシート資料を受領**（着手の前提）
2. 資料を下敷きにスキル項目を整理（追加・削除・更新）
3. レベル定義（例: 5段階）とフォーマットを策定
4. 社員に自己評価を実施
5. 社長レビュー・フィードバック → 改善
6. 本運用開始（定期更新サイクルを設計）

**汎用スキル項目の候補（資料受領前のたたき台）**

> ※ 単なるPC操作ではなく、**AI活用を前提とした現代的な汎用スキル**として設計する。
> AIにできる作業は積極的に任せ、人は判断と仕上げに集中することを基本姿勢とする。

### A. AI活用力（最重要・現代の前提スキル）

- **AI移譲の判断**: 面倒な作業・繰り返し作業をAIにやらせる発想
- **限界の見極め**: AIが苦手な作業（事実確認、最新情報、機微な判断等）を識別
- **役割分担の判断**: 「人がやるべきか / 工夫してAIにやらせるか」を場面ごとに選べる
- **プロンプト設計**: 目的を達成するための指示の出し方・分解の仕方
- **出力の検証**: AI出力を鵜呑みにせず、ファクト・品質を評価できる
- **ツール使い分け**: ChatGPT / Claude / Gemini / 画像生成系 / Claude Code 等を用途で選べる

### B. PC操作（基礎）

- ショートカット、タイピング速度、ファイル管理
- 複数アプリ・ウィンドウの切替効率

### C. 作業スピード・段取り

- 業務処理量、見積/段取り力
- AIに任せる前提でのタスク分解・並行処理

### D. ビジネス基礎

- 報連相、メール作法
- Slack / Chatwork 等のチャット運用

### E. ドキュメント作成

- Word、Excel、Google スプレッドシート
- Markdown（AIとの相性も良い）

### F. 情報収集力

- 検索スキル（Web、社内資料、AI検索）
- 一次情報を辿る習慣

### G. セキュリティ意識

- パスワード管理、機密情報取り扱い
- AIに渡してはいけない情報の判別

**次のアクション**

1. **【最優先】社長から過去のスキルシート資料を受領する**
2. 受領後、汎用スキル版のスキル項目をドラフト
3. 並行して、下記の確認事項を整理する

**確認事項（後ほど社長へ確認）**

| # | 内容 |
|---|---|
| 1 | 運用開始の希望時期 |
| 2 | 評価方式（自己評価のみ / 自己 + 社長レビュー） |
| 3 | 育成計画・給与査定との連動有無 |
| 4 | レベル定義の段階数（5段階 / 4段階 / 3段階 等） |
