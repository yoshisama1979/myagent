あなたは現在 **解析質問デッキ生成モード** で動作します。

解析系エージェント（HP分析ループ・ブログ分析ループ）が掲示板に積んでいる **社長への質問・判断待ち（🔔）** を集約し、社長がめくって答えるだけのプレゼン資料（HTML＋A4横PDF）として `site/decks/analysis-questions/` に生成する一発コマンドです。手動起動のみ（cron 化しない・社長決定 2026-07-15）。

## 引数（サイト指定・2026-07-15 追加）

- **引数なし** `/questions-deck` → **全サイト版**を `site/decks/analysis-questions/deck.html`（＋deck.pdf）に生成。
- **サイト指定** `/questions-deck <site> [<site>...]` → 指定サイトの掲示板（HP分析＋あればブログ）だけを対象に、**クライアント提出用のサイト別デッキ**を `site/decks/analysis-questions/<site>/deck.html`（＋deck.pdf）に生成。`<site>` は `site/hp-analysis/` 配下のディレクトリ名（例 ycom / yoshida / fujisaka）。存在しないサイト名なら生成せず、実在する候補を列挙して終了。

### サイト指定版＝クライアント提出用の書き方（2026-07-15 社長指示・必須）

サイト指定版は**クライアントに渡す資料**。読み手は**この資料以外何も見ていない前提**で、資料単体で完結させる：

| ルール | 中身 |
|---|---|
| **内部の仕組みを一切出さない** | `from-president.md`・`/hp-reply`・`/questions-deck`・Slack日報スレッド・mailbox・トークン・エージェント名（web-hanasaka 等）・掲示板/サイクル番号・他クライアント名は**書かない**。**内部管理番号（R-NNN・Q-NNN・B/T-NNN・PI-NNN 等）・内部URL/IP（`100.123.104.87` 等の Tailscale アドレス）・spec/archive 等の掲示板ページ名・`bin/` スクリプト名も同様に禁止**（資料内の通し番号 1〜N に振り直す）。実装側への言及が要るときは「制作担当」「弊社側」と言い換える |
| **回答方法は資料内で完結** | 「本資料の番号と選択肢でご回答ください（例：②は (a)）。いつもの連絡手段（メール・チャット）で担当まで。1件だけでも進みます」に統一 |
| **クライアント向け成果物ルール準拠**（CLAUDE.md） | です・ます調／専門用語（GA4・GSC・meta 等）には一言補足／ビジネスインパクトで語る |
| **中身は改変しない** | 質問の意図・数値・選択肢・固有名詞はそのまま（言い換えは表現だけ。事実を足したり丸めたりしない） |
| 表紙 | 発行元＝株式会社はなさか・対象サイト・発行日・件数。フッターの再生成コマンド表記は入れない |

全サイト版（引数なし）は**社長向けの内部資料**なので従来どおり（回答先パス・コマンド・エージェント名を書いてよい）。

## 必読ファイル

1. `.claude/rules/deck-format.md` — スライド資料の作り方（CSS骨格・PDF化コマンド・落とし穴・検証手順）。**デッキの技術面はすべてこれに従う**
2. `CLAUDE.md` — 行動指針（既読なら省略可）

## 対象の自動発見（サイト名を焼き込まない・最重要）

対象は **実行のたびにディレクトリ走査で発見**する。コマンドやデッキにサイト名の固定リストを持たない＝解析対象のサイトが増えれば次の生成から自動的に含まれる：

- `site/hp-analysis/*/index.html` （HP分析掲示板。ハブ `site/hp-analysis/index.html` 自身は除く）
- `site/hp-analysis/*/blog/index.html` （ブログ分析掲示板）

## 手順

### Step 1: 質問の収集（読み取りのみ）

発見した各掲示板の **🔔 セクション（社長への質問・判断待ち・連絡）** を読み、**未回答のものだけ**を拾う。各質問について取れる範囲で：

- 質問文・選択肢（あれば）
- いつからの質問か（Cycle 番号・日付・継続日数）
- 回答先（そのサイトの `from-president.md` のパス／Slack 日報スレッド返信／`/hp-reply`。掲示板の記載に従う）

**うち「Q番号・Cycle番号・回答先パス」は全サイト版（社長向け）にだけ載せる材料**。サイト指定版（クライアント提出用）では判断材料（質問・選択肢・ビジネス上の意味・実測数値）だけを使い、内部の番号・経路はデッキに出さない（上記「書き方」表）。

**質問の意図・固有事実（数値・固有名詞・選択肢）を改変しない**。要約は可だが、答えに影響する情報を落とさない。判断に迷う質問は原文寄りで載せる。あわせて各掲示板の「📥 分析に欲しい情報」も拾う（付録1枚に圧縮）。

### Step 2: デッキ生成（上書き再生成）

出力先（全サイト版＝`site/decks/analysis-questions/deck.html`／サイト指定版＝`site/decks/analysis-questions/<site>/deck.html`）を **毎回まるごと作り直す**（定期レポート＝スナップショット。生成日はタイトルに明記。過去版の保存はしない＝過去の質問履歴は各掲示板 archive が持つ）。deck-format.md の CSS 骨格（`@page { size: 297mm 210mm; margin: 0; }` トップレベル・`print-color-adjust: exact`・`table-layout: fixed`＋`colgroup`）に従う。外部 Web フォント・CDN に依存しない。

スライド構成の目安（**下記は全サイト版。サイト指定版は読み替える**＝表紙は「発行：株式会社はなさか＋対象サイト」、質問は資料内の通し番号 1〜N、「いつから／回答先」の列は載せず、回答方法スライドは「本資料の番号と選択肢で担当へ返信」の1枚にする）：

1. **表紙**：タイトル・生成日・合計件数（サイト数／質問数）
2. **サマリ**：サイト × 件数 × 最重要1件の一覧表。質問ゼロのサイトも「なし」と1行載せる（見落としでないことを示す）
3. **サイト別**：1サイト1枚目安（質問が多ければ分割・ゼロならスライドなし）。各質問は「質問文／選択肢／いつから／回答先」のカード or 表。**選択肢があるものは選択肢を必ず載せる**（社長が丸を付けるだけにする）
4. **回答方法**：どこにどう答えるかのまとめ1枚（from-president.md / Slack スレッド返信 / /hp-reply の使い分け）
5. **付録**：📥 分析に欲しい情報（全サイト分を1枚に圧縮・多ければ省略可）

### Step 3: PDF 化と検証

deck-format.md の実行コマンドで `deck.pdf` を生成し、**検証A**（MediaBox ≒ 842×595pt＝幅>高さ・ページ数一致）を通す。目視だけで「A4横」と報告しない。

### Step 4: ハブ登録

`site/decks/index.html` の資料一覧（`<!-- deck-rows -->` 直下）に本デッキの行が**あれば更新日・件数を更新、なければ追加**する（資料名：解析エージェントからの質問事項レポート。サイト指定版は「〃（<site>版）」として別行）。他の行は触らない。

### Step 5: 報告

社長に1〜3行で報告：合計質問数・最も古い/重い質問1件・**生成したデッキの閲覧URL**（全サイト版＝`http://100.123.104.87/decks/analysis-questions/deck.html`／サイト指定版＝`http://100.123.104.87/decks/analysis-questions/<site>/deck.html`。いずれも対応する PDF も併記）。

## 制約

| 掲示板・config・from-president の Read | ✅ 可（読むだけ） |

| 質問の創作・改変・推測での補完 | ❌ 不可（不明は原文のまま・「未確認」と明示） |
| 外部送信（Slack 投稿含む）・本番改変 | ❌ 不可（このコマンドは資料生成だけ。通知したければ社長が別途指示） |
| git commit / push | ❌ 不可（社長判断・別タイミング） |

## 重要

- **回答の受け皿は増やさない**：このデッキは既存の回答チャネル（from-president.md / Slack 返信）への**入口を1枚にまとめる読み取りビュー**であって、新しい台帳ではない。デッキに答えを書き込む運用にしない
- 質問ゼロの日でも生成してよい（「現在、未回答の質問はありません」の1枚＋サマリ）
- 気づきがあれば `data/hp-improve/skill-kaizen.md` に1行（無ければ「特記なし」）
