# Web開発用インストラクション

## CSS/Sass

- CSSはSassを使用しています
- CSSを変更する場合は、`css/sass/*.scss` ファイルを更新してください
- コンパイル済みのCSSファイルではなく、Sassのソースファイル（`.scss`）を編集します
- SCSS 自動コンパイル環境の構築は `rules/setup.md` を参照（初回セットアップ時のみ）

### ファイルの役割

- `base.scss`：プロジェクトをまたいで使える基本的なスタイル
- `common.scss`：プロジェクト内での共通スタイル
- `{ページ名}.scss`：ページごとのスタイル

### セレクタの指定方針

- **クラス名・IDを基本とする**  
  スタイルは原則としてクラス（`.class`）またはID（`#id`）で指定し、コンポーネントごとに明示的に当てる。

- **要素（タグ）セレクタのみの指定は避ける**  
  `ul`、`li`、`a`、`h1` など要素だけへの指定は行わない。どの要素に効いているか曖昧になり、意図しない上書きや詳細度の競合を招くため。

- **タグ指定は「本当に基本的なリセット・共通のみ」**  
  タグに直接指定する場合は、リセットやサイト全体のごく基本的な見た目に限定する。コンポーネント固有の見た目はクラス・IDで指定する。

- **タグのスタイルは上書きしない**  
  一度タグで基本スタイルを決めたら、別の場所でタグセレクタで上書きしない。コンポーネントごとの違いはクラスを付与し、そのクラスで指定する。

### CSS命名規則・ネスト構造
- BEMをベースにした命名規則を採用しています
- ネストは親要素のクラス名内に子要素を記述する形式です

**記述例:**
```scss
.parent {
  // 親要素のスタイル
  
  .parent__child {
    // 子要素のスタイル
    
    .child__child2 {
      // 孫要素のスタイル
    }
  }
}
```

**ルール:**
- ブロック名（`.parent`）を親とし、その中に要素名（`__child`）をネストして記述
- 要素の階層が深くなる場合も、直前の親要素のクラス名内にネストを継続。先祖クラス名は使用しない。クラス名が長くなりすぎないため。
- クラス名は`ブロック名__要素名`の形式を維持

### 修飾クラス（モディファイア）の書き方

`&` 自体は使用可能。ただし **BEM のモディファイア（`--xxx`）を `&--xxx` と短縮する書き方は禁止**。

#### ❌ BEM モディファイアの短縮は NG

理由: 検索時に `.item__label--sub` のような完全なクラス名でヒットさせるため。`&--sub` だと grep で見つけられない。

**悪い例:**
```scss
.item__label {
    font-size: 18px;

    &--sub {
        font-size: 15px;
    }
}
```

**良い例:**
```scss
.item__label {
    font-size: 18px;
}

.item__label--sub {
    font-size: 15px;
}
```

#### ✅ 複数クラス連結（`.item.active` 等）での `&` は使用可

要素にクラスが複数付いて状態を表現する場合などは `&` でネストしてよい。クラス名全体は変わらないため検索性に影響しない。

```scss
.item {
    color: #333;

    &.active {
        color: red;
    }

    &:hover {
        opacity: 0.8;
    }

    &::before {
        content: '';
    }
}
```

### メディアクエリの記述方法

スタイルはメディアクエリごとに分けて記述してください。

**記述例:**
```scss
#mainvisual{
    // PC用のスタイル
}

@media screen and (max-width: $mq_mobile){
    #mainvisual{
        // スマホ用のスタイル
    }
}
```

**ルール:**
- PC用のスタイルを先に記述
- その後に`@media screen and (max-width: $mq_mobile)`でスマホ用のスタイルを記述
- 同じセレクタをメディアクエリ内で再度記述し、必要なプロパティのみ上書きする

## HTML

- URLが指定されていない場合は、`#`としてください。

### レイアウト構造

#### `.section`
ページの中で大きな一つのまとまりとして縦に並ぶそれぞれの要素に使用します。

**使い方:**
- ページ内の主要なコンテンツエリアを`.section`で囲む
- 複数の`.section`を縦に並べてページを構成する
- レスポンシブ対応済み（スマホでは自動的にスタイルが調整される）

**修飾クラス:**
- `.section_first`：先頭のsectionに使用（上部の余白が広い）
- `.section_end`：末尾のsectionに使用（下部の余白が広い）
- `.section_odd`：背景色を変更したいsectionに使用
- `.section_tilt_l`：左に傾斜したデザインのsectionに使用
- `.section_tilt_r`：右に傾斜したデザインのsectionに使用

**子要素:**
- `.section__title`：セクションのタイトルに使用
- `.section__description`：セクションの説明文に使用
- `.section__block`：セクション内のブロック要素に使用
- `.section__group`：セクション内のグループ化された要素に使用（背景色と角丸が適用される）

#### `.container`
コンテンツの最大幅を制限し、中央揃えにするために使用します。

**使い方:**
- `.section`内に`.container`を配置して、コンテンツの幅を制限する
- 中央揃えで表示される
- レスポンシブ対応済み（スマホでは全幅表示になる）

## PHP定数

会社名や住所などの基本的な情報は`template/define.php`で定義されています。以下の定数が利用可能です：

- `COMPANY_NAME`：会社名、サイト名
- `COMPANY_ZIP`：郵便番号
- `COMPANY_ADDRESS`：住所（Googlemapで利用できるように建物名は入れない）
- `COMPANY_BUILDING`：建物名
- `COMPANY_FDIAL`：フリーダイヤル
- `COMPANY_TEL`：電話番号
- `COMPANY_FAX`：FAX番号
- `COMPANY_HOLIDAY`：定休日
- `COMPANY_TIME`：開館時間、受付時間など
- `COMPANY_MAIL`：メールアドレス（表示用）
- `COPY_RIGHT`：コピーライト表記
