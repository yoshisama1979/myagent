以下の手順を**一つずつ順番に**実行してください。
各ステップで何をしたかを報告すること。
**問題がある場合のみユーザーに確認を取って停止すること。**

プロジェクトルートは `project-config.md` を参照すること。

---

## ステップ1: ステージングと差分確認

変更したファイルをステージングする。
プロジェクトルートで `git diff --staged` を実行し、ステージングされた差分を確認する。

問題がある場合（バグ・仕様違反・セキュリティリスク・不要な変更）は問題点を指摘して**停止**。

問題がない場合はステップ1.5（Codex クロスレビュー）に進む。

---

## ステップ1.5: Codex クロスレビュー

ユーザー全体スコープのスラッシュコマンド `/codex-review` の手順に従い、
ステージング済みの差分を Codex（OpenAI GPT-5系）に投げてクロスレビューを実施する。

レビュー結果は次のステップでコミットメッセージ提案と一緒にユーザーに報告する。

**Codex の指摘の扱い:**
- 🔴 Critical（バグ・セキュリティ）が出た場合は、コミットメッセージ提案より先に
  問題点を強調してユーザーに報告し、対応するか・無視するかを確認する
- 🟡 Warning や 🟢 Suggestion は参考情報として併記し、ユーザーが判断する
- ✅ Approved の場合はその旨を簡潔に報告する

Codex MCP が利用不可の場合（接続エラー等）はその旨を伝えて、Claude のレビューのみで進める。

---

## ステップ2: 報告とコミットメッセージ提案

以下をまとめて報告し**停止**:

- ファイル一覧
- Claude による差分確認結果
- Codex クロスレビュー結果（ステップ1.5 の出力）
- コミットメッセージ提案

コミットメッセージは日本語で見出しと項目で提案すること。
過去のコミット履歴（`git log --oneline -5`）のフォーマットに合わせること。

ユーザーの承認を得てから次へ進む。

---

## ステップ3: コミット実行

承認されたコミットメッセージでコミットを実行し、結果を報告する。

---

## ステップ4: リファクタリング

まず `rules/refactoring.md` を読み込んでガイドラインを確認する。

プロジェクトルートで `git diff HEAD~1` を実行し、直前のコミットの差分を確認する。
ガイドラインに従って Claude 自身の評価でリファクタリング判断を行う。

### ステップ4.1: Codex クロスレビュー（条件付き）

ステップ1.5 の Codex レビュー結果を確認する:

- `critical` または `warnings` のいずれかが**非空**だった場合 → 続行
- 上記が空の場合（`suggestions` のみ・または完全に Approved）→ スキップしてステップ4.2 へ

**`suggestions` 単独では起動しない**（軽微な改善提案だけで Codex を再度呼ぶのはコストに見合わないため）。
ステップ1.5 で「ユーザーに参考情報として併記する」止まりだった指摘は、ステップ4.1 でも自動アクションは取らない。

続行する場合、`mcp__codex__codex` を以下のパラメータで呼び出す:

- `prompt`: `git diff HEAD~1` の内容と `rules/refactoring.md` の原則（重複コード・
  可読性・責務分離・命名・マジックナンバー・型安全性・パフォーマンス）を含める。
  「リファクタリング不要なら NO_ACTION、提案があれば具体的な箇所と内容を JSON で返す」と指示する
- `sandbox`: `read-only`
- `approval-policy`: `never`

Codex の提案を Claude 自身の評価と統合し、以下のいずれかで報告:

- 両者ともリファクタリング不要 → そのまま完了報告
- どちらかが提案 → 統合してユーザーに報告して**停止**
- 両者の判断が競合 → Claude の判断と Codex の指摘を並列に提示してユーザー判断を仰ぐ

### ステップ4.2: 実行と確認

リファクタリングを行った場合は、ガイドラインで指示されている通り関連するテストを実行して動作確認を行うこと。

リファクタリングが不要な場合はその旨を報告して完了。
リファクタリングを行った場合は、変更内容を報告して**停止**。ユーザーの確認後、ステップ1に戻って再度パイプラインを実行すること。
