よしだ歯科サイト 改善提案 — 実装エージェント引き渡し用

作成：2026-06-16 ／ 対象：yoshida-smile.info（WordPress）

これは サイトを構築・運用する実装エージェント向けの引き渡し資料。各提案は「着手できる粒度」で記述。社長⇄クライアント用の現状説明は 打ち合わせブリーフ を参照。

前提・方針（実装前に把握）

  • ゴール：来院（WEB予約・電話）の増加。運用方針は「アクセスは十分→質（来院・便利さ・わかりやすさ）の向上を優先」（2026-04-27 打合せ）。
  • 実データの裏付け：GA4で WEB予約 月127〜204件・電話タップ 月89〜153件のCVが発生。訪問者は減少（月7,026→4,725・約-33%）だが予約は増加＝CVR向上。来院に近い動線・ページの質を上げる方向が正しい。
  • データの注意：GSCは現状1日分（2026-06-14〜）＝「方向性の事実」として扱う。GA4の予約/電話はイベント発火数（「予約完了」の厳密定義は要確認）。
  • 合意フロー：本番改変は社長合意の上で。クライアント固有の事実（症例内容・成果数値）は捏造せず、無ければ「用意すべき内容」として社長経由で要求。

A. サイト実装タスク（実装エージェントが着手）

R-Y01：予約・WEB問診・電話の追従CTAを全ページに固定（SP最優先） 🔴

  • 観察：CVは電話タップ（GA4 `tel` 月89〜153）とWEB予約（`reservation` 月127〜204）に集中。WEB問診は導入決定済みだがサイト導線が未設置。予約アプリも導入予定。
  • なぜ：歯科サイトのCV＝予約・来院。スマホで「電話／WEB予約／WEB問診」が常に押せる状態が来院に直結。
  • 実装：WPテーマに追従CTA群を実装。SP＝画面下固定バー（電話 tel: リンク・WEB予約・WEB問診）、PC＝ヘッダー常時表示。WEB問診はサービス名/URL確定後に差し込めるようURLプレースホルダを用意。タップ計測（GA4イベント）も維持/付与。
  • 効果：高 ／ 工数：小〜中 ／ 優先度：🔴

R-Y02：高表示・低CTRページの title / meta description を最適化（来院意図ページ優先） 🔴

  • 観察：GSCで掲載1位でもクリック0のページが複数（例：「歯石と歯垢の違い」「歯垢と歯石の違い」、blog/?p=5246 は455表示・CTR0.66%）。来院に近いページは健闘：/cleaning/(1.0位)・/first/(2.0位)・/kidskyousei/(5.0位・33表示・0クリック)。
  • なぜ：検索結果でクリックされる文言になっていない。表示はあるので最小工数で効く。
  • 実装：WPのSEO設定（使用中プラグイン）で各ページの <title>・meta description を個別最適化。来院意図ページ（診療メニュー・/cleaning/・/first/・/kidskyousei/）を優先し、「クエリ語＋ベネフィット＋医院名＋箕面」を入れる。
  • 注意（事実と推測）：症状系（口内炎・歯石の違い等）は情報収集層で、CTRを上げても来院寄与は限定的（推測）。優先は来院意図ページ。症状系は R-Y05 で予約導線を拾う。
  • 効果：中 ／ 工数：小 ／ 優先度：🔴

R-Y03：箕面×診療メニューのローカルSEO受け皿を強化 🟡

  • 観察：主力KWは「箕面 歯医者」。地域×診療メニュー（矯正・ホワイトニング・小児・インプラント）が伸びしろ。/kidskyousei/ は表示あるが0クリック。
  • 実装：各診療メニューページに対応エリア・アクセス・地域名を明記、構造化データ（Dentist / LocalBusiness）を整備、関連ページへの内部リンク。※GBP（Googleビジネスプロフィール）最適化・レビュー獲得はサイト外＝B項（運用側）。
  • 効果：中〜高 ／ 工数：中 ／ 優先度：🟡

R-Y04：症例集ページの実装（WP） 🟡 ※運用管理ToDo（work_id 167）が期日超過（〜2026-05-30）で未完

  • 観察：信頼要素＝来院の決め手。医療系（YMYL）はE-E-A-Tにも効く。律速はクライアントからの症例内容の受領。
  • 実装：カスタム投稿タイプ「症例」を設計（フィールド：症状／処置内容／期間／費用目安／注意点・リスク／写真）。一覧・詳細テンプレートを実装し、クライアントが自分で更新できる運用に。内容受領前でも、データ構造とテンプレートは先行実装可能。受領しやすいよう上記フィールドの入力テンプレを用意して社長経由で渡す。
  • 効果：中〜高 ／ 工数：中 ／ 優先度：🟡

R-Y05：症状系ブログ記事 → 予約導線の内部リンク／CTA を共通挿入 🟢

  • 観察：症状系クエリ（口内炎・歯石の違い・歯茎の腫れ 等）で大量表示。情報収集層だが、記事内に予約/相談導線があれば一部を来院へ回収できる。
  • 実装：ブログ記事テンプレ末尾に「気になる症状は当院へご相談ください／WEB予約」の共通CTAブロックを挿入（R-Y01の予約導線と接続）。
  • 効果：中 ／ 工数：小 ／ 優先度：🟢

B. 運用・社長判断（実装エージェントの着手外・参考）

  • 広告（Google）：検索広告はCPA¥770・CV増で優秀＝単純増額は不要（Googleの増額推奨もなし）。休眠キャンペーン（土曜診療・ファインド・ディスプレイ＝支出¥0）の活用可否は広告運用側で判断。サイト実装タスクではない。
  • MEO / GBP：Googleビジネスプロフィール最適化・投稿運用・レビュー獲得はサイト外。対応範囲を社長がクライアントと握る（前回打合せの宿題）。
  • 外部リンク：「箕面 歯医者」向けに MISOKA / バイオガイアJAPAN への被リンク依頼（打診ルートはクライアント側）。
  • クライアント受領待ち：症例集の内容（R-Y04の律速）、WEB問診のサービス名/URL（R-Y01の差し込み先）。
  • 実装済み：Instagram埋め込み（GA4で月96クリックと利用確認）／WP本体アップグレード。
