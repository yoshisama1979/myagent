改善提案（詳細）

R-K01 🔴 全ページ meta description を固有化（技術/SEO・工数小）

観察：トップの meta description が「概要」2字（hp-audit）。実質未設定＝検索結果のスニペットが機能していない。なぜ問題か：検索結果でクリック理由を示せず CTR を落とす。改善案：各ページに固有の説明文（120字前後・クエリ＋ベネフィット）。例トップ：「ハワイ・ワイキキのコンドミニアムを日本人オーナーから直接レンタル。仲介手数料0円。予約・お問い合わせはこちら」。効果中〜高 ／ 工数小。

R-K02 🔴 ファーストビューでUSP明確化＋予約CTAボタン（CV/導線・工数中）

観察：PC fold はヒーロー写真に重なる低コントラストのUSP文が途中で切れ、装飾的なコピーが散らばる。fold内に押せる予約ボタンが無く電話番号のみ。SPはヘッダ導線は良いがCTAボタンは無し。なぜ問題か：最強の差別化（仲介手数料0円）が3秒で伝わらず、次の一手（予約/問い合わせ）への明確な入口が無い＝CV機会損失。改善案：FV上部に「オーナー直接契約だから仲介手数料0円」を高コントラストで1行＋「空室を確認・予約する」ボタン（→ ./contact/）を配置。SPはヘッダの TEL/MAIL を活かしつつ追従CTA。効果高 ／ 工数中。

R-K03 🟡 OGP/Twitter/canonical/JSON-LD を全ページ付与（技術/SNS・工数小〜中）

観察：OGP 0・Twitterカード 0・canonical 無し・JSON-LD 0（hp-audit）。なぜ問題か：旅行・宿泊は SNS/LINE で共有されやすいのに、共有時にタイトル画像が出ず訴求を落とす。canonical 無しは重複URLの評価分散リスク。改善案：og:title/description/image・twitter:card を全ページ、canonical を自己参照で付与。物件ページに LodgingBusiness 構造化データ。効果中 ／ 工数小〜中。

R-K04 🟡 h1整理・viewportズーム禁止解除・alt補完（構造/A11y・工数小）

観察：h1 が4個／viewport に maximum-scale=1.0, user-scalable=no（ズーム禁止）／img の alt 欠落4・空1。なぜ問題か：h1複数は主題が曖昧（SEO）。ズーム禁止はシニア層の可読性を損なう（旅行検討層と相性が悪い）。alt 欠落は画像SEOとA11yの損。改善案：ページ主題のh1を1つに、他は h2/装飾へ。viewport から maximum-scale/user-scalable を除去。物件・ギャラリー画像に内容を表す alt。効果中 ／ 工数小。

R-K05 🟡 各コンドミニアムページの情報充実（コンテンツ・工数中）

観察：物件5室（アイランドコロニー／ハワイアンモナーク／ワイキキバニアン／アクアマリーナ／ワイキキタウンハウス）はあるが、料金目安・最低宿泊数・定員/設備・予約の流れが薄い。ブログ最終更新2024-11で鮮度も落ちている。なぜ問題か：宿泊予約は「いくら・何人・どう予約」が決め手。情報不足は離脱と問い合わせ前の不安を生む。改善案：各物件に「料金目安・定員・設備・最低宿泊・予約の流れ・FAQ・周辺地図」を表で整理（読みやすく）。固有数値は捏造せず Q-K03 で社長/クライアント確認。効果中〜高 ／ 工数中。
