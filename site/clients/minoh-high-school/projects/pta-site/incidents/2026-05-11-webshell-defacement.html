1. エグゼクティブサマリ

2026年5月11日、minokopta.com の複数ページがインドネシア系オンラインギャンブルサイト「PAPADEWA」を宣伝するスパムページに置き換えられていることを発見しました。

調査の結果、原因は同一サーバアカウントで運用していた別サイト konjaku-photo.com 配下に2026年1月13日頃に設置されていた webshell（遠隔操作用の不正PHPファイル）5件であることが判明しました。さくらのレンタルサーバは共有FTPユーザー権限のため、konjaku 配下に置かれた webshell から minokopta のファイルも書き換え可能な状態にありました。

当日中に以下を完了しました：

  • 改ざんファイル6件の復旧（4件は正規ファイルで上書き、2件は不要ファイルとして削除）
  • webshell 5件の無効化（3件は使用していなかった WordPress 削除に伴い物理削除、2件はリネームによる隔離）
  • サーバパスワードおよび WordPress 管理者パスワードの変更
  • 未使用の WordPress（konjaku/blog/）の削除による攻撃面縮小

復旧後の表示確認・スパム残存チェックともに問題なく、攻撃の継続的経路はすべて遮断されています。

2. 何が起こったか

2.1 改ざんの内容

minokopta.com の6ページが、PAPADEWA というインドネシア系オンラインギャンブルサイトを宣伝するページに置き換えられていました。

# 改ざん対象 サイズ（改ざん後） 改ざん日時
1mail_about/index.html517,436 byte4/25 17:06
2template/index.html542,254 byte5/3 14:01
3terms/index.html541,873 byte5/3 23:49
4topics/index.html542,030 byte5/4 12:10
5contact/index.html541,677 byte5/4 12:10
6contents/3_KOUHOU/index.html542,962 byte5/4 12:11

改ざんページの主な特徴:

  • <html lang="id-ID">（インドネシア語）で記述
  • title: PAPADEWA Situs Resmi Hari Ini Tempat Nikmati Pengalaman Terbaik!
  • canonical タグで minokopta.com の各URLを正規ページとして固定
  • amphtml タグで AMP代替として攻撃者のCloudflare Pages ドメイン（minokopta-ppdw.pages.dev）に誘導
  • Cloudflare Insights ビーコンによるアクセス計測

これは「Japanese SEO Spam」「Indonesian Gambling Spam」として知られる手口で、被害サイトのSEO評価に寄生してギャンブルサイトを検索結果に表示させ、最終的に攻撃者のドメインへ誘導することを目的としています。

2.2 攻撃の目的

PHP実行を伴わない静的HTML置換型のSEOスパム改ざんで、被害サイト訪問者を直接攻撃するものではなく、検索エンジン経由でのユーザー誘導を狙ったものです。

3. 原因

3.1 侵入経路

調査の結果、minokopta.com 自体への直接侵入ではなく、同一サーバアカウントで運用されていた konjaku-photo.com 配下に設置されていた webshell が攻撃の足場でした。

さくらのレンタルサーバでは複数サイトを同一OSユーザー権限で動かす構造のため、konjaku 配下の webshell から minokopta のファイルも書き換え可能な状態にありました。

3.2 設置されていた webshell（全5件）

すべて konjaku-photo.com 配下に設置されていました。

# 設置先 サイズ 種類
1konjaku/vendor/phpmailer/phpmailer/language/phpmailer.lang-ke.php184,830 byte主webshell
2konjaku/blog/wp-content/plugins/jetpack/json-endpoints/class.wpcom-json-api-list-lamp-endpoint.php184,830 byte冗長化バックドア
3konjaku/blog/wp-content/themes/twentythirteen/content-media.php184,830 byte冗長化バックドア
4konjaku/blog/wp-includes/Text/Diff/Engine/index.php184,830 byte冗長化バックドア
5konjaku/php_lib/Library/securimage/shellfinder1.php4,136 byteライバル攻撃者排除ツール

ファイル1〜4 は完全に同一の webshell（184,830バイトの同一バイト数）で、攻撃者は1つの webshell を異なる4ヶ所にコピー配置していました。これは「1ヶ所が削除されても他の場所から再活動できる」ようにする冗長化バックドアの典型的な手口です。

特徴:

  • ファイル先頭に "PHP Melody" というアプリケーションのコメントを偽装ヘッダとして配置
  • 周囲のファイル群とパーミッションを変えている（644 vs 周囲の604）ため、-perm 644 での検索で機械的に検出可能
  • タイムスタンプは touch で過去日付に偽装（2022年・2024年の日付）

5番目の shellfinder1.php は「SCR1PT K1DD135 Finder v1.0 | Created by XNY」と名乗るインドネシア語の不正ツールで、他の攻撃者が設置したwebshellを検出・削除する機能を持つもの。攻撃者が乗っ取ったサーバを他の攻撃者と共有しないために設置する典型的なツールです。

3.4 webshell 設置を許した根本原因

konjaku-photo.com アプリケーション自体の脆弱性が原因と推定されますが、本日の対応時間内では根本原因の特定には至っていません。後日の追加調査が必要な事項です。

候補:

  • konjaku-photo.com アプリケーション自体のアップロード機能等の脆弱性
  • vendor/ ディレクトリが Web 公開されていた設定ミス
  • PHPMailer 等の依存ライブラリの既知脆弱性（CVE-2016-10033 等）放置
  • 認証情報（FTP/SSH/コントロールパネル）の漏洩

4. 対応内容

Phase 1: 状況把握と侵入経路特定（調査）

  1. SSH接続して minokopta 配下のディレクトリ構造を確認
  2. ファイルタイムスタンプから改ざんされたファイル群を特定
  3. 改ざんファイル内容からスパム改ざんと判明（PAPADEWA, lang="id-ID"）
  4. grep -rl 'lang="id-ID"' で改ざんファイル6件を網羅的に特定
  5. アクセスログから攻撃者IP（162.128.185.123）と侵入経路（konjaku経由）を特定
  6. 複数の検索手法（eval系コード、ファイルサイズ、特定文字列、ファイル名パターン）でwebshell5件を特定

Phase 2: 証拠保全

~/incident_20260511/evidence/ 配下に以下を保全:

  • webshells/ - webshell 5ファイルのコピー
  • minokopta_hacked/ - 改ざんされた minokopta の6ファイルのコピー
  • attacker_logs/ - 攻撃者IPのアクセスログ（空のまま、必要に応じて後日抽出）

Phase 3: webshell の無効化

webshell 5ファイルすべてを .QUARANTINED 拡張子にリネームし、PHPとして実行不能な状態に変更。これにより、攻撃者が引き続き webshell にアクセスしても何も起きない状態にしました。

Phase 4: 認証情報の更新

以下のパスワードを変更しました:

  • サーバパスワード（SSH/FTP用）
  • WordPress 管理者パスワード（konjaku, minokopta 両方）

Phase 5: 改ざんファイルの復旧

オリジナルファイル（手元の保管していたもの）で改ざんされた6ファイルを上書き復旧。

  • terms/index.html → 9,233 byte（復旧後）
  • topics/index.html → 14,998 byte（復旧後）
  • mail_about/index.html → 5,256 byte（復旧後）
  • contents/3_KOUHOU/index.html → 12,326 byte（復旧後）
  • contact/index.html → もともと存在しないファイルだったため削除
  • template/index.html → もともと存在しないファイルだったため削除

Phase 6: 攻撃面の縮小

使っていなかった konjaku/blog/（WordPress）ディレクトリを丸ごと削除しました。これに伴い、その配下にあった webshell 3件も物理的に削除されました。残った webshell 2件は隔離状態を維持しています。

Phase 7: 復旧確認

  • 全6ファイルが正規サイズ（数KB〜十数KB）で配置されていることを確認
  • grep による全文検索で lang="id-ID", PAPADEWA, minokopta-ppdw がいずれも検出されないことを確認
  • find -size 184830c および grep "PHP Melody" で webshell の残存ゼロを確認

Phase 8: 副次的な問題への対処

konjaku-photo.com アプリのSQLログが php_error.log に約41.6GB蓄積していたため、当該ファイルをゼロクリア。ディスク容量を解放しました（根本のログ出力停止は後日対応事項）。

7. 教訓と所感

7.1 学んだこと

マルチサイト共有環境のリスク

さくらのレンタルサーバは複数サイトを同一OSユーザー権限で動かす構造のため、1つのサイトの脆弱性が他のサイトに波及します。今回は使っていない konjaku 配下のアプリの脆弱性から、稼働中の minokopta が改ざんされる結果になりました。

未使用サイトの放置リスク

最も深刻な点として、攻撃の足場になった konjaku-photo.com の /blog/（WordPress）は使用していませんでした。使っていないからメンテナンスもされず、脆弱性が放置され、結果として攻撃の温床になりました。「使っていないなら削除する」が最も効果的な対策です。

冗長化バックドアの認識

攻撃者は1つの webshell が見つかっても他から再活動できるよう、複数箇所に同一の webshell をコピー配置します。発見した1つを削除しただけでは不十分で、必ず網羅的に探す必要があります。複数の検索手法（コードパターン、ファイルサイズ、特定文字列、ファイル名、パーミッション）を組み合わせることが重要でした。

7.2 防御の有効性

検出手段

  • ファイル更新時刻の確認 → 改ざんファイル群の特定に有効
  • 特定文字列の grep（PHP Melody, phpsugar 等） → webshell の網羅的検出に有効
  • ファイルサイズによる絞り込み → 同一バイト数のコピー webshell の検出に有効
  • パーミッションによる絞り込み → 攻撃者のファイルとアプリのファイルの区別に有効

ログの価値

過去3ヶ月分のアクセスログがあったことで、攻撃者IPの特定・攻撃タイムラインの再構築・侵入経路（別サイト経由）の特定がすべて可能になりました。ログ保管は事後調査において決定的に重要です。
