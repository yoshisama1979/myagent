## 運用メモ
- 改善版draft が無い B → `wp-draft.py post`（新規下書き作成・タイトルに「【改善版】」）→ id を上表に記録。以降は `wp-draft.py update --id <改善版draft id>`。
- **元記事の post id は編集系に渡さない**（読み取り `get` のみ）。`update` は draft 限定ガードで公開済みを拒否。
- 状態：✍️改善版作成済（レビュー待ち）／🔁更新済／✅本番反映済（人間が元記事に反映）／⏳事実待ち。
- 固有事実が要る箇所は `【要・社長確認】` のまま残し、社長回答後に反映。
