# 財務データ保管ルール

経営判断のための財務データを置く場所。**`.gitignore` で除外しているため、コミットされない。**

## 配置

```
data/financial/
├── YYYY/                             # 年度データ（YYYY = 開始年、例: 2025 → FY2025 = 2025-04 〜 2026-03）
│   ├── pl.csv                        # 損益計算書（年度全体）
│   └── bs.csv                        # 貸借対照表（期末時点）
└── YYYY-MM/                          # 月次データ
    ├── pl-monthly.csv                # 月次推移表（PL推移）
    └── ledger-by-client.csv          # 補助元帳（取引先別）
```

※ 日本の会計では「2025年度」は 2025-04 〜 2026-03 を指す。フォルダ名は開始年に揃える。

## マネーフォワードからのエクスポート手順

1. マネーフォワード クラウド会計にログイン
2. **会計帳簿 → 残高試算表**（推奨）または レポート → 該当のレポートを開く
3. **期間を指定**（年度通年なら 4/1〜翌3/31、期中なら 4/1〜直近月末）
4. 「CSV出力」または「エクスポート」をクリック
5. 出力されたCSVを `data/financial/YYYY/`（年度通年）または `data/financial/YYYY-MM/`（月次）に配置

CSVヘッダーに「開始月：YYYY-MM-DD」「終了月：YYYY-MM-DD」が含まれており、`site/business/annual-forecast.php` はそこから経過月数を自動判定する。

## 取得対象（決算書ベース）

| # | CSV種類 | 何が見える | 性質 | 頻度 |
|---|--------|---------|-----|------|
| 1 | 損益計算書（PL） | 当月の売上・費用・利益 | 期間のフロー（経営成績） | 月次 |
| 2 | 貸借対照表（BS） | 月末時点の資産・負債・純資産・キャッシュ | 時点のストック（財務状況） | 月次 |
| 3 | 月次推移表（PL推移） | 売上・経費・利益の月別推移 | 時系列分析（伸びの把握） | 月次 |
| 4 | 補助元帳（取引先別） | クライアント別の売上・入金 | 顧客指標（依存度） | 月次 or 四半期 |

## セキュリティ

- 生データは **コミットしない**（.gitignore で除外済み）
- 集計結果は `business/reviews/YYYY-MM.html` や `business/kpi.html` に **数値のみ** 記載してOK
- 個別取引明細・取引先名の詳細は集計結果からは除外する（数値のみで議論）

## 分析の流れ

1. 社長が月初にマネーフォワードから CSV をエクスポート → 本ディレクトリに配置
2. AI（私）が `bin/analyze-mf.py` を実行して集計
3. 結果を `business/reviews/YYYY-MM.html` の「2. KPI 実績」セクションに記入
4. KPI トレンドを `business/kpi.html` に蓄積（将来的）

## 分析スクリプト

`bin/analyze-mf.py` で CSV を読み込み、Markdown 形式のレポートを出力。

```
python bin/analyze-mf.py data/financial/2025/
```

出力内容（2026-05-16 時点）:
- 損益計算書サマリ：売上・販管費・各種利益・売上比
- 販管費上位（売上比1%以上）
- 貸借対照表サマリ：資産・負債・純資産・自己資本比率
- 売掛金 取引先別（年間回収額 上位）

CSV のエンコーディングは Shift_JIS（マネーフォワード標準）。スクリプト側で自動処理。
